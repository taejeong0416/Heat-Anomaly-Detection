## Phase별 핵심 결과

### Phase 0. 데이터 통합 및 기본 정제

- 60개 월별 CSV를 단일 parquet으로 통합 (16,868,714행)
- 총사용량 = 1~24시 합으로 재계산, `유형` 컬럼 drop (전 기간 `-`)
- 파생 컬럼 추가: 연도, 월, 일, 요일, 계절, 난방시즌, 공휴일
- 설비 종별 변경 0건 확인 (전 기간 일관)
- **이상값은 EDA 후 판단으로 이월**

### Phase 1. EDA

- 종별 간 스케일 격차 확인 → **종별 분리 분석 필수** 확정
- 난방시즌/비난방시즌 패턴 차이 극명, 업무용·공공용 평일/주말 차이 두드러짐
- 음수값 6,604건(0.04%), NaN 포함 209,894건(1.2%), NaN 30%+ 설비 45개
- 지사 익명화로 기온 미사용 결정 → 월/계절로 대체
- **Phase 2 전처리 파라미터 전량 확정**
- `전처리방침.md`, `도메인조사.md` 작성

### Phase 2. 정밀 전처리

- 16,868,714행 → 16,511,109행 (97.9% 보존)
- 품질 검증 전 항목 PASS (음수 0, NaN 0, 제거율 <3%, 보간 0.52%)
- 종별별 시간당 99.9%ile threshold: 주택용 2.59, 업무용 2.74, 공공용 4.38 Gcal
- NaN=24 분포: Branch_0007 최다(12,611건) → DCU 장애 추정
- 정규화 파라미터(종별×활성시즌 8그룹: 난방 6 + 냉수용 2) 산출 저장 (`norm_params.json`)

## 이상 유형 분류 체계

> 상세: `이상유형정의.md` 참조

### 1계층: 데이터 이상 (규칙 기반, 우선순위 계층)

모델 전단 필터. 해당 조건에 걸리면 이상유형 라벨을 부여하고 모델에 넣지 않는다.

| 우선순위 | 유형 | 조건 | 학습 시 |
|:--------:|------|------|:-------:|
| 1 | 검침기리셋 | `any(hour < 임계치)` ⚠미확정 | 행 제거 |
| 2 | 총량음수 | `total < 0` | 행 제거 |
| 3 | 다중음수이상 | `neg_count >= 3` | 행 제거 |
| 4 | 극단값이상 | `any(hour > threshold × 100)` | 행 제거 |
| 5 | 데이터수집이상 | `nan_count >= 3` | 행 제거 |

### 2계층: 이상사용 (모델 기반)

데이터 이상 필터를 통과한 데이터에 대해 정상 패턴 모델로 탐지. Phase 5~6에서 확정.

| 유형 후보 | 패턴 | 실무적 의미 |
|----------|------|------------|
| 급증형 | 전일 대비 급등 | 배관 누수, 밸브 고장 |
| 야간 이상형 | 업무용 야간/주말 비정상 사용 | 밸브 미폐쇄, HVAC 오류 |
| 장기 미사용 후 급증형 | 연속 제로 → 급증 | 열량계 재작동 |
| 계절 역행형 | 비활성시즌 고사용 (난방: 4~10월, 냉수: 10~5월) | 열손실, 배관 누수 |
| 주말/공휴일 이상형 | 업무용 주말 ≈ 평일 | 무인 건물 HVAC 미해제 |
| 기저 유량 이상형 | 최소 시간대 지속 높음 | 배관 누수 |
| 패턴 이탈형 | 클러스터 중심 거리 상위 | 설비 오작동, 용도 변경 |
