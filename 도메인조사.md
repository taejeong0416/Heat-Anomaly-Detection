> 이상사용 탐지 프로젝트의 도메인 지식 기반. 전처리·Feature Engineering·이상 유형 정의의 근거로 활용.

## 1. 열량계 측정 원리

### 1-1. 열량 계산식

```
열량(Gcal) = 유량(m³) × (공급온도 - 환수온도)(°C) × 비열계수
```

- 유량 측정: 초음파식(transit time difference) 또는 기계식(임펠러 회전수)
- 온도 측정: 공급/환수 배관에 각각 온도센서 설치
- 적산 방식: 순시 열량을 시간 단위로 누적(적산)하여 저장

### 1-2. 열량계 유형

| 유형 | 원리 | 장점 | 단점 |
|------|------|------|------|
| 초음파식 | 순방향/역방향 초음파 전파 시간차로 유속 측정 | 마모 없음, 고정밀 | 공기 유입 시 오류, 불순물 >0.5%에서 오차 증가 |
| 기계식 (임펠러) | 터빈 회전수로 유량 측정 | 단순, 저렴 | 저유량 미검출, 마모에 의한 정밀도 저하 |

### 1-3. 법적 규정

- **적산열량계 법정 검정 유효기간: 5년** (계량에 관한 법률 시행령 별표 13)
- 재검정 유효기간도 5년
- 본 데이터 기간(2021~2025)이 정확히 5년 → 데이터 내에 **열량계 교체 이벤트 포함** 가능

## 2. 한국 지역난방 시스템

### 2-1. 공급 구조

```
열병합발전소 (CHP)
  ↓ 1차측 배관 (약 115~120°C 고온수)
건물 기계실
  ↓ 판형 열교환기 (1차/2차 분리)
  ↓ 2차측 배관
각 세대
  ↓ 온돌 (바닥 복사난방) + 급탕
```

- **간접 연결 방식**: 1차측(발전소)과 2차측(건물 내부) 배관이 열교환기로 분리. 대부분의 한국 지역난방이 채택.
- 열교환기를 거친 후 발전소 측 물은 온도가 낮아져 회수, 건물 측 물은 온도가 높아져 세대로 공급.

### 2-2. 열량계 설치 단위

| 종별 | 설치 단위 | 비고 |
|------|-----------|------|
| 주택용 | **세대별** (아파트 각 호) | 유량계 또는 열량계. 단위: m³/ton(유량계) 또는 MWh/Mcal/Gcal(열량계) |
| 업무용 | **건물별** 또는 사용자별 | 건물 기계실 1차측 열량계 |
| 공공용 | 건물별 | 업무용과 유사 |
| 냉수용 | 건물별 | 지역냉방 냉수 별도 계량 |

### 2-3. 운영 지사

- KDHC는 수도권 중심으로 지사 운영 (본 데이터: 19개 지사)
- 지사별 차이 원인:
  - **열원 용량**: 지사별 발전 용량 차이 (예: 동탄지사 800MW)
  - **공급 지역 특성**: 신도시(아파트 위주) vs 구도시(상업/공공 혼재)
  - **배관 노후도**: 설치 시기에 따른 열손실 차이
  - **기후 미세차**: 같은 수도권이라도 지역별 외기온 편차 존재

## 3. 정상 사용 패턴

### 3-1. 주택용

한국 아파트의 **온돌(바닥 복사난방)** 특성: 열관성(thermal mass)이 크다. 5시간 연속 온수 공급 열량으로 하루 간헐 난방 가능.

| 시간대 | 패턴 | 설명 |
|--------|------|------|
| 00~06시 | 저~중 | 취침 중 온도 낮추거나 유지. 바닥 축열로 서서히 냉각 |
| 06~09시 | 중~고 | 기상 후 실내 온도 회복 수요 |
| 09~17시 | 저 | 외출 세대는 1~2단 낮추어 유지 (KDHC 권장: 완전 OFF보다 유지가 효율적) |
| 17~22시 | **고 (피크)** | 귀가 후 온도 올리는 피크 시간대 |
| 22~24시 | 중 | 취침 준비, 약간 낮춤 |

- 평일/주말 차이: **상대적으로 작음** (온돌 연속난방 특성)
- KDHC 권장: 간헐난방(껐다 켰다)보다 **연속난방**이 난방비 절약에 유리

### 3-2. 업무용

| 시간대 | 패턴 | 설명 |
|--------|------|------|
| 00~06시 | 최저 | 야간 최소 유지 (동파 방지) |
| 07~08시 | 급증 | **예열(pre-heating)**: 업무 시작 1~2시간 전 가동 |
| 09~18시 | **고 (피크)** | HVAC 스케줄 가동 시간 |
| 18~22시 | 급감 | 퇴근 후 감소 |
| 22~24시 | 최저 | 야간 최소 유지 |

- 평일/주말 차이: **매우 큼** (주말 사용량 현저 감소)
- HVAC가 건물 에너지 소비의 약 54.9% 차지

### 3-3. 공공용

- 업무용과 유사한 패턴이나, 공공시설 운영 시간에 따라 다소 차이
- 주말/공휴일: 공공시설 휴관 시 사용량 감소

### 3-4. 냉수용 (지역냉방)

- 흡수식 냉동기로 3~4°C 냉수 생산하여 건물에 공급
- **하절기(6~8월)에만 사용 집중** — 난방과 정반대 계절 패턴
- 주로 업무용/상업용 대형 건물에 적용
- KDHC 일부 지역(중앙지사, 고양 등)에서만 운영

출처:
- [서울에너지공사 열사용요령](https://www.i-se.co.kr/mass101)
- [THE RADIANT FLOOR HEATING SYSTEM IN KOREA (IBPSA)](https://publications.ibpsa.org/proceedings/bs/1999/papers/bs1999_PA-25.pdf)

## 4. 계절 및 기온 특성

### 4-1. 난방 시즌 정의

| 구분 | 기간 | 열 공급 방식 |
|------|------|-------------|
| 연속난방 | 11월 ~ 4월 말 | 24시간 열 공급 |
| 간헐난방 | 5월 ~ 10월 | 외기온 25°C 이하 또는 장마철에만 공급 |
| 열사용 비수기 | 4/23 ~ 9/22 (KDHC 기준) | |

**본 프로젝트 정의**: 난방시즌 = 11~3월 (연속난방 핵심 기간)

### 4-2. 난방도일(HDD)

- **Heating Degree Day**: 일평균 기온이 기준온도(18°C) 이하인 날에 대해 `(18 - 일평균기온)`을 누적 합산
- HDD가 클수록 난방 에너지 수요 많음
- 한국에너지공단에서 전국 기준 냉난방도일 정보 공개

> **본 프로젝트**: 지사 익명화로 지역별 기온 매칭 불가 → HDD 대신 월/계절 변수로 대체

### 4-3. 요금 시즌 구분 (KDHC)

| 시즌 | 월 | 요금 특성 |
|------|-----|----------|
| 동절기 | 12~2월 | 최고 사용량, 최고 요금 |
| 춘추절기 | 3~5월, 9~11월 | 중간 |
| 하절기 | 6~8월 | 최저 난방 (급탕/냉방 위주) |

출처:
- [한국에너지공단 냉난방도일 (공공데이터포털)](https://www.data.go.kr/data/15093745/openapi.do)
- [KDHC 열사용 요금](https://www.kdhc.co.kr/)

## 5. 검침 데이터 이상의 원인

### 5-1. 음수값 발생 원인

| 원인 | 메커니즘 | 데이터 특징 | 빈도 |
|------|----------|------------|------|
| **역류(Backflow)** | 고객측 압력 > 공급측 → 난방수 역방향. 야간 수요 감소, 배관 정비 중 발생 | 소규모 음수, 총사용량은 양수 가능 | 중간 |
| **검침기 교체/리셋** | 적산값 0 초기화 → 이전값과 차이가 극단적 음수 | 수천만 Gcal급 음수 1회 | 낮음 |
| **적산값 롤오버** | 적산값이 최대 표시값 초과 → 0으로 복귀 | 교체와 유사한 패턴 | 극히 낮음 |
| **통신 오류/보정** | 패킷 손실/타이밍 오류로 적산값 비순차 수신 | 간헐적 음수 + 이후 양수 스파이크 | 중간 |
| **온도센서 역전** | 비난방기에 공급온도 < 환수온도 (미세 유량 존재 시) | 소규모 음수 장기 지속 | 낮음 |

출처:
- [AMI & Negative Meter Readings (Safe-T-Cover)](https://www.safe-t-cover.com/blog/ami-negative-meter-readings-reveal-backflow-water-quality-issues)
- [Three years of hourly data from 3021 smart heat meters (Nature Scientific Data)](https://www.nature.com/articles/s41597-022-01502-3)

### 5-2. NaN/결측 발생 원인

| 원인 | 패턴 | 빈도 |
|------|------|------|
| **통신 장애** | 1~수 시간 결측, 특정 지역/시간대 집중 | 가장 흔함 |
| **열량계 배터리 소진** | 특정 설비에서 장기간 전 시간대 결측 | 중간 |
| **열량계 고장** | 갑자기 발생, 교체 전까지 지속 | 중간 |
| **열량계 교체 작업** | 교체 당일 수 시간 결측 + 적산값 리셋 | 낮음 |
| **집중기(DCU) 장애** | 해당 구역 전체 결측 (다수 설비 동시) | 낮음 |

### 5-3. 열량계 고장 유형

| 고장 유형 | 데이터 영향 | 실무적 의미 |
|-----------|------------|------------|
| **멈춤(배터리/부품)** | 장기 제로 또는 장기 NaN | 미계량 → 공동 난방비 전가 |
| **과다계량(센서 오류)** | 갑작스런 사용량 급증 | 난방비 폭탄 |
| **과소계량(마모)** | 지속적 저사용량 | 다른 세대 대비 비정상적 저사용 |
| **유량계/열량계 혼동** | 약 160% 과다 요금 | 설치 유형 확인 필요 |

### 5-4. 배관/설비 이상

| 이상 | 데이터 특징 |
|------|------------|
| **배관 누수** | 미사용 시에도 24시간 일정한 기저 사용량 |
| **밸브 미폐쇄** | 비난방 시간/시즌에 소량 사용 지속 |
| **열교환기 fouling** | 점진적 효율 저하 → 같은 실내온도 유지에 더 많은 열량 필요 |

## 6. 원격검침(AMI) 통신 시스템

### 6-1. 통신 방식

- **기존**: KS-PLC(유선) 단일방식
- **2021년~**: IP-PLC(유선) + LTE(무선) 표준화 → 음역지역 해소
- HPGP, Wi-Sun 등 다양한 무선 통신 방식 도입

### 6-2. 데이터 수집 주기

- 시간별(hourly), 일별, 월별, 연별로 적산열량계에 기록
- 원격으로 수집하여 중앙 시스템에 저장
- 원격 장애진단 및 소프트웨어 업데이트 가능
