## Phase 4. 정상 패턴 모델링

> "무엇이 정상인가"를 먼저 정의해야 이상을 판단할 수 있다.
> GMM(Gaussian Mixture Model)으로 정상 패턴 프로토타입을 정의하고, log-likelihood를 이상 점수로 활용한다.

### 4-0. 클러스터링 전처리

- [x] 종별×시즌 분리 (8그룹: 난방 3종별×2시즌 + 냉수용×2시즌)
- [x] 제로 사용일 제외 (총사용량=0 → hour_ratio=NaN, 패턴 정의 불가)
- [x] 피처 선택: `hour_ratio_1~24` (24차원, 패턴 형태)
  - hour_ratio는 합≈1로 정규화되어 있으므로 추가 스케일링 불필요
  - 비식별화 가중치와 무관 (비율이므로 스케일 불변)

### 4-1. GMM 클러스터링

> K-Means 대비 장점: 타원형 클러스터 허용, 확률적 할당(soft clustering), BIC 기반 모델 선택, log-likelihood가 곧 이상 점수.

- [x] 종별×시즌별 GMM 학습
  - covariance_type='full', reg_covar=1e-6
  - 샘플 학습 (그룹별 15만 행) → 전체에 predict
- [x] n_components 선택: BIC 기준 (n=2~8 탐색, 냉수용 2~4)
  - 결과: 난방 6그룹 전부 k=8, 냉수용_냉방시즌 k=2, 냉수용_비냉방시즌 k=4
  - k=8 천장 효과 인지하나, 팀원 환경에서 학습 완료된 결과를 유지 (재학습 비용 대비 이득 불명확)
- [ ] 전체 데이터에 predict → cluster_id, gmm_log_likelihood ← **미완료** (모델 pkl은 존재)
- [x] 클러스터별 24시간 평균 곡선 시각화 → "정상 패턴 프로토타입"
- [ ] 제로/극소 사용일: cluster_id=-1, gmm_log_likelihood=NaN

### 4-2. Context Z-score

> 횡단면 비교: "같은 조건의 다른 건물들 대비 얼마나 다른가."
> LightGBM 대신 그룹 통계 사용 — 이상값에 robust(중위수/MAD), 학습/검증 분리 불필요, 해석 명확.

- [ ] 그룹 정의: 종별 × 지사 × 월 × 요일
- [ ] 그룹별 총사용량 중위수(median) + MAD 산출
- [ ] context_zscore = (실제 - median) / MAD
  - 양수 = 같은 조건 건물 대비 과다 사용
  - 음수 = 같은 조건 건물 대비 과소 사용
- [ ] 그룹 통계 JSON 저장 (테스트 데이터 적용용)

## Phase 5. 이상 탐지

### 5-1. 이상 유형 정의 (EDA + 도메인 조사 반영)

> EDA 결과 + 도메인 조사(`도메인조사.md` §5)를 바탕으로 유형을 구체화. Phase 4 결과 후 최종 확정.

| # | 유형 | 탐지 조건 | 도메인 근거 | 실무적 의미 |
|---|------|-----------|------------|------------|
| 1 | 급증형 | 전일 대비 변화율 상위 N% + context_zscore 상위 | 열량계 과다계량 고장(센서 오류), 배관 누수 시작, 적산값 롤오버 | 현장 점검(열량계 교체/배관 점검) |
| 2 | 야간 이상형 | 야간 비율이 종별·월 그룹 평균 + 2σ 이상 | 업무용은 야간 동파 방지 최소 유지만 정상. 야간 지속 = 밸브 미폐쇄, HVAC 스케줄 오류 | 에너지 낭비 점검 |
| 3 | 패턴 이탈형 | GMM log-likelihood 하위 N% 또는 AE 복원 오차 상위 | 정상 패턴(주택: 아침·저녁 피크, 업무: 주간 집중)에서 벗어난 형태 | 설비 오작동, 용도 변경 |
| 4 | 장기 미사용 후 급증형 | 난방시즌 내 연속 제로 N일 → 다음날 총사용량 > threshold | 열량계 멈춤(배터리 소진) 후 교체·재작동, 공실 후 입주 | 열량계 고장 여부 확인 |
| 5 | 계절 역행형 | 비활성시즌에 사용량 상위 N% | 난방: 비난방기 고사용, 냉수: 비냉방기 고사용 | 배관/열교환기 점검 |
| 6 | 주말/공휴일 이상형 | 업무용·공공용 주말 사용량 ≈ 평일 수준 | 무인 건물 HVAC 미해제, 에너지 낭비 | 운영 관리 점검 |
| 7 | 기저 유량 이상형 | 최소 시간대 사용량이 종별·시즌 기준 대비 지속적으로 높음 | 배관 누수 시 미사용 시간에도 24시간 일정한 기저 유량 발생 | 배관 누수 점검 |

### 5-2. Isolation Forest

- [ ] Feature: 24시간 사용량 + 파생 Feature + context_zscore (**gmm_log_likelihood 제외** — Phase 4 모델 출력을 IF 입력에 넣으면 정보 누수)
- [ ] 종별별 별도 학습 (종별 간 스케일·패턴 차이). 시즌 분리는 미적용 (종별 분리로 충분, 시즌은 피처로 학습)
- [ ] contamination 파라미터: 0.01~0.05 범위에서 그리드 탐색 (GMM weak label 기반 F1 최대화로 선정, 한계 인지)
- [ ] anomaly score 산출

### 5-3. Autoencoder 복원 오차

- [ ] 구조: 24차원 입력 → Dense(16) → Dense(8) → Dense(16) → 24차원 출력
- [ ] 종별별 별도 학습
- [ ] reconstruction error 산출 → 상위 N%를 이상으로 판정
- [ ] IF + AE + MP 3개 앙상블: 2개 이상 이상 → 고신뢰, 1개만 → 저신뢰

### 5-4. 통계 기반 보조 탐지

- [ ] 종별·월·지사 그룹 내 z-score 기반 필터링 (지사 포함)
- [ ] IQR 기반 극단치 탐지 (종별·지사·월 그룹, Tukey 1.5×IQR)

### 5-5. Matrix Profile (시계열 자기 참조 이상 탐지)

> 기존 방법(GMM/AE/IF)은 "다른 설비·정상 패턴 대비" 비교. MP는 "자기 자신의 과거 대비" 비교 → 관점 차별화.

- [ ] 설비별 일별 총사용량 시계열 구축 (설비당 ~1,800일)
- [ ] `stumpy.stump(ts, m=7)` — 주간 패턴 window로 Matrix Profile 계산
  - mp_score = nearest neighbor distance (값이 클수록 이질적 구간)
  - 상수 시계열(std≈0), 14일 미만 설비 → 스킵
- [ ] 종별별 weak label 기반 F1 그리드 탐색 (0.01~0.05) → mp_flag 산출
- [ ] 앙상블 통합: IF + AE + **MP** 3개 중 2개 이상 → 고신뢰 (다수결)
- [ ] Phase 6 패턴이탈형 규칙에 mp_score 추가 (gmm_ll OR ae_score OR mp_score)
