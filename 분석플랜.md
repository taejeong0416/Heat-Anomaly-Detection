## Phase 4. 정상 패턴 모델링

> "무엇이 정상인가"를 먼저 정의해야 이상을 판단할 수 있다.
> GMM(Gaussian Mixture Model)으로 정상 패턴 프로토타입을 정의하고, log-likelihood를 이상 점수로 활용한다.

### 4-0. 클러스터링 전처리

- [x] 종별×시즌 분리 (8그룹: 난방 3종별×2시즌 + 냉수용×2시즌)
- [x] 제로 사용일 제외 (총사용량=0 → hour_ratio=NaN, 패턴 정의 불가)
- [x] 피처 선택: `hour_ratio_1~24` (24차원, 패턴 형태)
  - hour_ratio는 합≈1로 정규화되어 있으므로 추가 스케일링 불필요
  - 비식별화 가중치와 무관 (비율이므로 스케일 불변)

### 4-1. GMM 클러스터링

> K-Means 대비 장점: 타원형 클러스터 허용, 확률적 할당(soft clustering), BIC 기반 모델 선택, log-likelihood가 곧 이상 점수.

- [x] 종별×시즌별 GMM 학습
  - covariance_type='full', reg_covar=1e-6
  - 샘플 학습 (그룹별 15만 행) → 전체에 predict
- [x] n_components 선택: BIC 기준 (n=2~8 탐색, 냉수용 2~4)
  - 결과: 난방 6그룹 전부 k=8, 냉수용_냉방시즌 k=2, 냉수용_비냉방시즌 k=4
  - k=8 천장 효과 인지하나, 팀원 환경에서 학습 완료된 결과를 유지 (재학습 비용 대비 이득 불명확)
- [ ] 전체 데이터에 predict → cluster_id, gmm_log_likelihood ← **미완료** (모델 pkl은 존재)
- [x] 클러스터별 24시간 평균 곡선 시각화 → "정상 패턴 프로토타입"
- [ ] 제로/극소 사용일: cluster_id=-1, gmm_log_likelihood=NaN

### 4-2. Context Z-score

> 횡단면 비교: "같은 조건의 다른 건물들 대비 얼마나 다른가."
> LightGBM 대신 그룹 통계 사용 — 이상값에 robust(중위수/MAD), 학습/검증 분리 불필요, 해석 명확.

- [ ] 그룹 정의: 종별 × 지사 × 월 × 요일
- [ ] 그룹별 총사용량 중위수(median) + MAD 산출
- [ ] context_zscore = (실제 - median) / MAD
  - 양수 = 같은 조건 건물 대비 과다 사용
  - 음수 = 같은 조건 건물 대비 과소 사용
- [ ] 그룹 통계 JSON 저장 (테스트 데이터 적용용)

## Phase 5. 이상 탐지

### 5-1. 이상 유형 정의 (Phase 4 클러스터 + 4-agent review 결과 — 2026-05-13 확정)

> 상세 규칙·우선순위·의사결정 근거는 `이상유형정의.md` 참조.

**우선순위 (재정렬)**:
```
장기미사용후급증 > 계절역행 > 야간이상 > 주말이상 > 급증 >
        기저유량 > 연속가동 > 간헐사용 > 패턴이탈 (catch-all)
```

| # | 유형 | 탐지 조건 | 실무적 의미 |
|---|------|-----------|------------|
| 1 | 장기미사용후급증형 | 활성시즌 & **설비별 직전 7일 연속 총사용량=0** & 당일 ma7_ratio 상위 20% | 열량계 멈춤 후 재작동, 공실 후 입주 |
| 2 | 계절역행형 | 비활성시즌 & 총사용량 **종별별 임계** 이상 (주택 99%, 나머지 95%) | 열손실, 배관 누수, 설비 오작동 |
| 3 | 야간이상형 | **전 종별 적용** & night_ratio ≥ 종별·월 정상 평균 + **종별별 σ** (주택 +3σ, 나머지 +2σ) | 밸브 미폐쇄, HVAC 스케줄 오류 |
| 4 | 주말이상형 | 업무·공공용 & 주말·공휴일 & 총사용량 ≥ 정상 평일 중위수 × 0.8 | 무인 건물 HVAC 미해제 |
| 5 | 급증형 | ma7_ratio 상위 10% AND \|context_zscore\| 상위 10% | 배관 누수, 밸브 고장, 과다계량 |
| 6 | 기저유량이상형 | baseload ≥ 종별·활성시즌 95%분위 | 배관 누수, 상시 가동 설비 |
| 7 | 연속가동형 (신규) | hourly_std ≤ 종별·활성시즌 정상 5%분위 AND baseload ≥ 정상 80%분위 | 보일러 상시 가동, HVAC 24시간 작동, 누수 지속 유량 |
| 8 | 간헐사용형 (신규) | cv ≥ 종별·활성시즌 정상 95%분위 AND hourly_std ≥ 정상 95%분위 | 열량계 간헐 결함, 비정형 거주 패턴, 급수 결함 |
| 9 | 패턴이탈형 (catch-all) | **gmm_log_likelihood 하위 5% AND (ae_score 상위 5% OR mp_score 상위 5%)** + 잔여 흡수 → 미분류 0% | 설비 오작동, 약한 비전형 신호 |

**Phase 4 클러스터 검토에서 발견된 사항**:
- 주택용 새벽 가동(C0 34.9%) 정상 클러스터 → 주택용 야간이상은 +3σ로 엄격화하되 적용은 유지
- 주택용 평탄 클러스터(C4 41.6%)의 더 극단판이 미분류로 누락 → **연속가동형 신규 추가**
- Phase 6b 미분류 50건 검토 결과 cv z=+2.01, ae z=+3.11 → **간헐사용형 신규 추가**
- 패턴이탈 OR 결합 기존 정의 → 50~70% 흡수되어 다른 유형 가려짐 → **AND 결합 (5% AND 5%)** 으로 강화 후 catch-all 역할 부여

### 5-2. Isolation Forest

- [ ] Feature: 24시간 사용량 + 파생 Feature + context_zscore (**gmm_log_likelihood 제외** — Phase 4 모델 출력을 IF 입력에 넣으면 정보 누수)
- [ ] 종별별 별도 학습 (종별 간 스케일·패턴 차이). 시즌 분리는 미적용 (종별 분리로 충분, 시즌은 피처로 학습)
- [ ] contamination 파라미터: 0.01~0.05 범위에서 그리드 탐색 (GMM weak label 기반 F1 최대화로 선정, 한계 인지)
- [ ] anomaly score 산출

### 5-3. Autoencoder 복원 오차

- [ ] 구조: 24차원 입력 → Dense(16) → Dense(8) → Dense(16) → 24차원 출력
- [ ] 종별별 별도 학습
- [ ] reconstruction error 산출 → 상위 N%를 이상으로 판정
- [ ] IF + AE + MP 3개 앙상블: 2개 이상 이상 → 고신뢰, 1개만 → 저신뢰

### 5-4. 통계 기반 보조 탐지

- [ ] 종별·월·지사 그룹 내 z-score 기반 필터링 (지사 포함)
- [ ] IQR 기반 극단치 탐지 (종별·지사·월 그룹, Tukey 1.5×IQR)

### 5-5. Matrix Profile (시계열 자기 참조 이상 탐지)

> 기존 방법(GMM/AE/IF)은 "다른 설비·정상 패턴 대비" 비교. MP는 "자기 자신의 과거 대비" 비교 → 관점 차별화.

- [ ] 설비별 일별 총사용량 시계열 구축 (설비당 ~1,800일)
- [ ] `stumpy.stump(ts, m=7)` — 주간 패턴 window로 Matrix Profile 계산
  - mp_score = nearest neighbor distance (값이 클수록 이질적 구간)
  - 상수 시계열(std≈0), 14일 미만 설비 → 스킵
- [ ] 종별별 weak label 기반 F1 그리드 탐색 (0.01~0.05) → mp_flag 산출
- [ ] 앙상블 통합: IF + AE + **MP** 3개 중 2개 이상 → 고신뢰 (다수결)
- [ ] Phase 6 패턴이탈형 규칙에 mp_score 추가 (gmm_ll OR ae_score OR mp_score)

## Phase 7. 알고리즘 성능 평가

> 비지도 환경(ground truth 없음)에서 알고리즘의 정량적 성능을 보이기 위한 평가 단계.
> 단순 베이스라인(±30% 룰)과 비교하여 우리 알고리즘의 유의미한 향상을 입증한다.

### 7-1. Synthetic Anomaly Injection

- [ ] 정상 데이터(`is_anomaly==0` & `총사용량>0`)에서 종별별 표본 추출 (1,000건)
- [ ] 7유형 이상을 인위적으로 주입 (종별 × 유형당 200건):
  - 급증형: 사용량 ×3.5
  - 야간이상형: 22~06시에 평균 ×4 가산
  - 패턴이탈형: 24시간 시간 셔플
  - 장기미사용후급증형: 대부분 0 + 짧은 구간 급증 ×6
  - 계절역행형: 비활성시즌 행에서 전반적 ×2 + 가산 (활성시즌 패턴 모방)
  - 주말이상형: 주말·공휴일 행에서 ×1.5 + 평탄화 (업무용/공공용 한정)
  - 기저유량이상형: 모든 시간대에 일정 기저 가산 (배관 누수 시뮬)
- [ ] 단일 행 피처 재계산: hour_ratio, cv, night/day_ratio, peak_hour, baseload, zero_count, hour_diff_*
- [ ] 학습된 IF/AE 모델로 점수 산출 → 앙상블 플래그 (IF OR AE, recall 우선)

### 7-2. 베이스라인 비교

- [ ] **현행 운영 방식**: "같은 설비의 전년·전전년 동일 일자 평균 총사용량 대비 |편차|/평균 > 0.30 → 이상"
- [ ] 전년·전전년 이력이 없을 시 종별·월 중위수로 fallback
- [ ] Precision / Recall / F1 산출 (전체 + 유형별 Recall)

### 7-3. 정성 비교 표

| 항목 | Ours | Baseline ±30% |
|------|:----:|:-------------:|
| 탐지 가능 유형 수 | 9유형 | 1유형 (사용량 편차) |
| 야간/패턴/시계열/컨텍스트 반영 | O | X |
| 해석 가능성 | O (SHAP) | △ |
| 유형 자동 분류 | O | X |

### 7-4. 수동 검토 표본 (Phase 6 결과 stratified)

- [ ] Phase 6 분류 결과에서 유형별 7건씩 추출 (≈ 49건)
- [ ] 도메인 전문가 재검토용 CSV 생성

### 7-5. 평가 한계 (보고서 본문에 명시)

- **합성 이상은 실제 이상의 근사**: 도메인 케이스를 완전히 커버하지 않음. 실측 이상에서의 성능과 일치 보장 불가
- **history 의존 피처 미재계산**: ma7_ratio / autocorr_lag1 / mp_score는 단일 행 주입으로 재계산이 어려워 원본값 유지 (보수적 추정)
- **In-sample 평가**: 학습 모델이 동일 데이터(2021~2025)를 이미 학습했으므로 unseen 데이터 일반화 성능과 동일하지 않음
- **임계 가정**: IF top 2%, AE top 3%, contamination 0.02는 학습 단계 결정값 차용
- **베이스라인 단순화**: ±30% 룰은 실무 운영에서의 통상적 단순 기준이며, 더 정교한 baseline(이전 기간 대비, IQR 등)과의 비교는 향후 과제
