## Phase 1. 데이터 탐색 (EDA)

> 정제된 통합 데이터를 기반으로 탐색. 이후 단계의 의사결정 근거를 마련한다.

### 1-1. 기초 통계 확인

- [ ] 연도별·월별 행 수 추이 (설비 증감 파악)
- [ ] 종별(주택용/업무용/공공용/냉수용) 비율 및 분포

### 1-2. 사용량 분포 확인

- [ ] 종별별 시간대 평균 사용 곡선 (1~24시) → 종별 간 패턴 차이 확인
- [ ] 총사용량 분포: 히스토그램, percentile (99%, 99.9%)
- [ ] 종별별 극단값 분포 확인 (음수, 상위 percentile)
  - **결과에 따라**: 종별별 오류값 제거 threshold 확정
- [ ] 사용량=0 비율: 월별 추이 → 비난방 시즌(여름) 제로 비율 확인
  - **결과에 따라**: 여름 데이터를 분석 대상에서 제외할지, 포함하되 별도 처리할지 결정

### 1-3. 시계열 패턴 확인

- [ ] 주택용/업무용/공공용 각각 대표 건물 1~2개 선정 → 1년간 일별 총사용량 추이 시각화
- [ ] 평일 vs 주말 사용 패턴 비교 (종별별)
- [ ] 월별(계절별) 사용량 변화 → 난방 시즌 vs 비난방 시즌 구분 기준 도출
- [ ] NaN 분포: 특정 시간대/설비에 집중되는지 확인
  - **결과에 따라**: 결측치 정밀 처리 전략(보간/제거/설비 단위 제외) 확정

### 1-4. 외부 변수 활용 가능성 판단

- [ ] 지사가 익명화되어 있으므로 지역별 기온 매칭이 가능한지 검토
  - 방법 A: 전국 평균 기온 사용 (단순하지만 지역 차이 무시)
  - 방법 B: 지사별 사용량 패턴으로 지역(기후대) 역추정 시도
  - 방법 C: 기온 변수 사용하지 않고 월/계절로 대체
  - **결과에 따라**: 기온 변수 포함 여부 및 수집 방법 확정

### 1-5. EDA 산출물

- [ ] EDA 요약 노트북 (`notebooks/02_eda.ipynb`)
- [ ] 종별별 시간대 평균 곡선 차트
- [ ] 위 확인 결과 기반으로 Phase 2~6 세부 파라미터 확정

## Phase 2. 정밀 전처리

> EDA 결과를 반영하여 분석 가능한 최종 형태로 데이터를 정제한다.

### 2-1. 오류값 threshold 재조정

- [ ] EDA에서 확인한 종별별 99.9% percentile 기반으로 극단값 threshold 확정
- [ ] Phase 0에서 보수적으로 처리한 값 재검토 및 재적용
- [ ] 제거 전후 비교 기록

### 2-2. 결측치 정밀 처리

- [ ] Phase 0에서 보류한 NaN(행 내 1~12개) 처리
  - NaN ≤ 3개: **시간 보간** (해당 행 내 앞뒤 시간대 선형보간)
  - NaN 4~12개: **해당 행 제거** 또는 설비 단위 동일 요일 평균으로 보간 (EDA 결과 기반 결정)
- [ ] 설비 단위 NaN 누적률 확인
  - 특정 설비가 전체 기간의 30% 이상 NaN → 분석 대상에서 제외 검토
- [ ] 처리 후 `총사용량` 재계산

### 2-3. 정규화

- [ ] 종별 단위로 Min-Max 또는 Z-score 정규화
- [ ] 정규화 전 원본값 보존 (별도 컬럼 또는 별도 파일)
- [ ] 정규화 파라미터(min/max 또는 mean/std) 저장 → 결과 해석 시 역변환용

### 2-4. 산출물

- [ ] 최종 정제 데이터: `data/processed/all_data_clean.parquet`
- [ ] 정밀 전처리 코드: `notebooks/03_preprocessing_v2.ipynb`

## Phase 3. Feature Engineering

### 3-1. 기본 Feature (24차원)

- 1시~24시 시간별 사용량 (정규화 후)

### 3-2. 파생 Feature

| Feature | 산출 방법 | 목적 |
|---------|-----------|------|
| 총사용량 | 1~24시 합 | 전체 규모 |
| peak 시간 | argmax(1~24시) | 주요 사용 시간대 |
| 야간 비율 | (22~06시 합) / 총사용량 | 야간 이상 탐지 |
| 주간 비율 | (09~18시 합) / 총사용량 | 업무시간 패턴 |
| 시간별 표준편차 | std(1~24시) | 사용 변동성 |
| 전일 대비 변화율 | (당일 총 - 전일 총) / 전일 총 | 급증 탐지 |
| N일 이동평균 대비 비율 | 당일 총 / MA(7일) | 단기 추세 대비 |
| 연속 제로일 수 | 직전 총=0 연속 일수 | 장기 미사용 탐지 |

### 3-3. 컨텍스트 Feature

| Feature | 출처 |
|---------|------|
| 요일 (0~6) | 날짜에서 파생 |
| 공휴일 여부 | holidays 라이브러리 |
| 월 / 계절 | 날짜에서 파생 |
| 종별 (원핫 또는 레이블) | 원본 컬럼 |
| 기온 (선택) | Phase 1-4 결과에 따라 결정 |

### 3-4. Feature Engineering 산출물

- [ ] Feature 생성 코드 (`notebooks/04_features.ipynb`)
- [ ] Feature 포함 데이터 저장

## Phase 4. 정상 패턴 모델링

> "무엇이 정상인가"를 먼저 정의해야 이상을 판단할 수 있다.

### 4-1. 종별별 Clustering

- [ ] 종별(주택용/업무용/공공용/냉수용)별로 분리하여 각각 클러스터링
- [ ] K-Means 적용 (k=3~8, silhouette score로 최적 k 선정)
- [ ] 각 클러스터의 24시간 평균 곡선 시각화 → "정상 패턴 프로토타입"

### 4-2. 기대 사용량 모델 (Context Model)

- [ ] 입력: 종별, 요일, 월, (기온) → 출력: 기대 총사용량
- [ ] 방법: 동일 조건 그룹 평균 (단순) 또는 LightGBM 회귀
  - EDA 결과에 따라 선택
- [ ] Residual 계산: `실제 - 기대` → 이후 이상 탐지의 핵심 feature

### 4-3. 산출물

- [ ] 클러스터별 정상 패턴 프로토타입 차트
- [ ] Context Model 학습 결과 및 Residual 분포
- [ ] 코드 (`notebooks/05_normal_pattern.ipynb`)

## Phase 5. 이상 탐지

### 5-1. 이상 유형 최종 정의

> Phase 1, 4의 결과를 바탕으로 **데이터 기반** 유형 정의를 확정한다.

**후보 유형 (EDA 결과에 따라 추가/삭제/수정)**:

| # | 유형 | 탐지 조건 (초안) | 확정 시점 |
|---|------|------------------|-----------|
| 1 | 급증형 | 전일 대비 변화율 상위 N% + Residual 상위 | Phase 3 후 threshold 확정 |
| 2 | 야간 이상형 | 야간 비율이 종별 평균 + 2σ 이상 | Phase 1에서 종별 야간 패턴 확인 후 |
| 3 | 패턴 이탈형 | 클러스터 중심과의 거리 상위 N% 또는 AE 복원 오차 상위 | Phase 4 클러스터링 후 |
| 4 | 장기 미사용 후 급증형 | 연속 제로 N일 이상 → 다음날 총사용량 > threshold | Phase 1에서 제로 패턴 확인 후 N 결정 |
| 5 | 계절 역행형 | 비난방 시즌에 사용량 상위 N% | Phase 1에서 계절 구분 확정 후 |

### 5-2. Isolation Forest

- [ ] Feature: 24시간 사용량 + 파생 Feature + Residual
- [ ] 종별별 별도 학습 (종별 간 스케일·패턴 차이 때문)
- [ ] contamination 파라미터: 0.01~0.05 범위에서 탐색
- [ ] anomaly score 산출

### 5-3. Autoencoder 복원 오차

- [ ] 구조: 24차원 입력 → Dense(16) → Dense(8) → Dense(16) → 24차원 출력
- [ ] 종별별 별도 학습
- [ ] reconstruction error 산출 → 상위 N%를 이상으로 판정
- [ ] Isolation Forest 결과와 앙상블: 둘 다 이상 → 고신뢰, 하나만 → 저신뢰

### 5-4. 통계 기반 보조 탐지

- [ ] 종별·월별 그룹 내 z-score 기반 필터링
- [ ] IQR 기반 극단치 탐지

### 5-5. (선택) Matrix Profile

- [ ] 설비 단위로 일별 총사용량을 시계열로 연결
- [ ] `stumpy`로 discord(가장 이질적 구간) 탐지
- [ ] 데이터 규모에 따라 샘플링 적용 여부 결정

### 5-6. 산출물
