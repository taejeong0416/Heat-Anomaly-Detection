## Phase 2. 정밀 전처리

> EDA 결과 + 도메인 조사(`도메인조사.md`)를 반영하여 분석 가능한 최종 형태로 데이터를 정제한다.
> 상세 근거는 `전처리방침.md` 참조.

### 2-0. 대상 데이터 범위

- [x] 분석 대상: **주택용 / 업무용 / 공공용 / 냉수용** 4개 종별
- [x] 냉수용: 시즌 정의 반전(난방시즌→냉방시즌 6~9월)하여 동일 파이프라인 적용

### 2-1. Phase 1: 음수 처리 (PNNL Section 2.4)

> 근거문헌: Fowler et al. (2015) PNNL-24331

- [x] **[1단계] 총량음수 행 제거**: sum(24h) < 0 → 행 제거 (PNNL Table 2-2)
- [x] **[2단계] 인접양수 상쇄**: ±2h 윈도우, Case 1/2/3 (PNNL Section 2.4.1)
- [x] **[3단계] 무상쇄 음수 NaN**: 잔여 음수 → NaN 변환 (PNNL Section 2.4.2)

### 2-2. Phase 2: 결측 처리 (PNNL Section 3)

- [x] **[4단계] NaN ≥ 3 일자 제거**: elbow 분석 기반 (PNNL Section 2.3)
- [x] **[5단계] 연속 NaN 평가**: 연속1=보간, 연속≥2=제거 (PNNL Section 3 Option 2)

### 2-3. Phase 3: 설비 필터

- [x] **[6단계] 1년 미만 설비 제거**: Quesada et al. (2024)
- [x] **[7단계] 결측률 28%+ 설비 제거**: 분포 단절점 분석
- [x] **[8단계] 처리비율 30%+ 설비 제거**: PNNL Section 2.3 + 분포 분석

### 2-4. Phase 4: 극단값 처리

- [x] **[9단계] 종별 시간당 99.9%ile × 100 이상 행 제거 + clip**

### 2-5. 후처리

- [x] 총사용량 재계산
- [x] **종별·활성시즌** Z-score 정규화 파라미터 산출 (8그룹)
- [x] 정제 데이터는 원본 스케일 유지, 파라미터 JSON 저장

## Phase 3. Feature Engineering

> 도메인 지식(`도메인조사.md`)에 기반하여 열 사용 패턴의 이상을 포착할 수 있는 feature를 설계한다.

### 3-1. 기본 Feature (24차원)

- 1시~24시 시간별 사용량 (정규화 후)

### 3-2. 파생 Feature

| Feature | 산출 방법 | 목적 | 도메인 근거 |
|---------|-----------|------|------------|
| 총사용량 | 1~24시 합 | 전체 규모 | |
| peak 시간 | argmax(1~24시) | 주요 사용 시간대 | 주택용 17~22시, 업무용 09~18시가 정상 피크 |
| 야간 비율 | (22~06시 합) / 총사용량 | 야간 이상 탐지 | 업무용은 야간 거의 없어야 정상 (동파 방지 최소 유지만). 야간 지속은 밸브 미폐쇄 또는 HVAC 스케줄 오류 |
| 주간 비율 | (09~18시 합) / 총사용량 | 업무시간 패턴 | 업무용·공공용 HVAC 가동 시간대. 이 비율이 낮으면 패턴 이탈 |
| 시간별 표준편차 | std(1~24시) | 사용 변동성 | 정상 설비는 시간대별 변동이 일정 범위 내. 극단적 변동은 검침 오류 가능 |
| 전일 대비 변화율 | (당일 총 - 전일 총) / 전일 총 | 급증 탐지 | 열량계 과다계량 고장, 배관 누수 시작 시점 |
| N일 이동평균 대비 비율 | 당일 총 / MA(7일) | 단기 추세 대비 | 온돌 열관성으로 정상 사용량은 서서히 변화. 급변은 이상 |
| 연속 제로일 수 | 직전 총=0 연속 일수 | 장기 미사용 탐지 | 난방시즌 제로 = 열량계 멈춤/공실. 비난방시즌 제로 = 정상 가능 |
| 주말 대비 비율 | 당일 총 / 동일 설비 주말 평균 | 주말 이상 탐지 | 업무용·공공용 주말 고사용 = 무인 건물 가동, 에너지 낭비 |
| 기저 사용량 | min(1~24시) | 최소 사용 수준 | 배관 누수 시 미사용 시간에도 일정한 기저 유량 발생 |

### 3-3. 컨텍스트 Feature

| Feature | 출처 | 비고 |
|---------|------|------|
| 요일 (0~6) | 날짜에서 파생 | 업무용·공공용 평일/주말 차이 반영 |
| 공휴일 여부 | holidays 라이브러리 | 공휴일은 주말과 유사한 패턴 기대 |
| 월 / 계절 | 날짜에서 파생 | HDD 대용 (지사 익명화로 기온 직접 매칭 불가) |
| 난방시즌 여부 | 11~3월 | KDHC 열사용 비수기(4/23~9/22) 참고하되 단순화 |
| 냉방시즌 여부 | 6~9월 | 냉수용 전용. 난방시즌의 반전 개념 |
| 종별 (원핫 또는 레이블) | 원본 컬럼 | |
| 지사 (레이블) | 원본 컬럼 | 지사별 열원 용량·건물 구성·배관 노후도 차이 반영 |

> **기온 변수 미사용** — 지사 익명화로 지역별 HDD 매칭 불가. 월/계절이 HDD의 proxy 역할.

## Phase 4. 정상 패턴 모델링

> "무엇이 정상인가"를 먼저 정의해야 이상을 판단할 수 있다.

### 4-1. 종별별 Clustering

- [ ] 종별(주택용/업무용/공공용/냉수용)별로 분리하여 각각 클러스터링
  - 냉수용: 냉방시즌(6~9월) 데이터로 학습. 시즌 정의만 반전, 파이프라인 동일.
- [ ] K-Means 적용 (k=3~8, silhouette score로 최적 k 선정)
- [ ] 난방시즌/비난방시즌 분리 클러스터링 검토 (EDA에서 패턴 차이 극명)
- [ ] 각 클러스터의 24시간 평균 곡선 시각화 → "정상 패턴 프로토타입"

### 4-2. 기대 사용량 모델 (Context Model)

> 도메인 근거: 열 사용량은 외기온(HDD), 건물 특성, 운영 스케줄에 따라 결정. 기온 대신 월/계절을 proxy로 사용.

- [ ] 입력: 종별, 지사, 요일, 월, 난방시즌, 공휴일 → 출력: 기대 총사용량
- [ ] 방법: LightGBM 회귀 (EDA에서 지사·종별·계절 복합 효과 확인)
  - 비교 대상: 동일 조건 그룹 평균 (baseline)
- [ ] Residual 계산: `실제 - 기대` → 이후 이상 탐지의 핵심 feature
  - Residual이 큰 양수 = 기대보다 과다 사용 (누수, 과다계량, 밸브 고장)
  - Residual이 큰 음수 = 기대보다 과소 사용 (열량계 멈춤, 과소계량)

## Phase 5. 이상 탐지

### 5-1. 이상 유형 정의 (EDA + 도메인 조사 반영)

> EDA 결과 + 도메인 조사(`도메인조사.md` §5)를 바탕으로 유형을 구체화. Phase 4 결과 후 최종 확정.

| # | 유형 | 탐지 조건 | 도메인 근거 | 실무적 의미 |
|---|------|-----------|------------|------------|
| 1 | 급증형 | 전일 대비 변화율 상위 N% + Residual 상위 | 열량계 과다계량 고장(센서 오류), 배관 누수 시작, 적산값 롤오버 | 현장 점검(열량계 교체/배관 점검) |
| 2 | 야간 이상형 | 야간 비율이 종별·월 그룹 평균 + 2σ 이상 | 업무용은 야간 동파 방지 최소 유지만 정상. 야간 지속 = 밸브 미폐쇄, HVAC 스케줄 오류 | 에너지 낭비 점검 |
| 3 | 패턴 이탈형 | 클러스터 중심 거리 상위 N% 또는 AE 복원 오차 상위 | 정상 패턴(주택: 아침·저녁 피크, 업무: 주간 집중)에서 벗어난 형태 | 설비 오작동, 용도 변경 |
| 4 | 장기 미사용 후 급증형 | 난방시즌 내 연속 제로 N일 → 다음날 총사용량 > threshold | 열량계 멈춤(배터리 소진) 후 교체·재작동, 공실 후 입주 | 열량계 고장 여부 확인 |
| 5 | 계절 역행형 | 비활성시즌에 사용량 상위 N% | 난방: 비난방기 고사용, 냉수: 비냉방기 고사용 | 배관/열교환기 점검 |
| 6 | 주말/공휴일 이상형 | 업무용·공공용 주말 사용량 ≈ 평일 수준 | 무인 건물 HVAC 미해제, 에너지 낭비 | 운영 관리 점검 |
| 7 | 기저 유량 이상형 | 최소 시간대 사용량이 종별·시즌 기준 대비 지속적으로 높음 | 배관 누수 시 미사용 시간에도 24시간 일정한 기저 유량 발생 | 배관 누수 점검 |

### 5-2. Isolation Forest

- [ ] Feature: 24시간 사용량 + 파생 Feature + Residual
- [ ] 종별별 별도 학습 (종별 간 스케일·패턴 차이)
- [ ] 난방시즌/비난방시즌 분리 학습 검토
- [ ] contamination 파라미터: 0.01~0.05 범위에서 탐색
- [ ] anomaly score 산출

### 5-3. Autoencoder 복원 오차

- [ ] 구조: 24차원 입력 → Dense(16) → Dense(8) → Dense(16) → 24차원 출력
- [ ] 종별별 별도 학습
- [ ] reconstruction error 산출 → 상위 N%를 이상으로 판정
- [ ] Isolation Forest 결과와 앙상블: 둘 다 이상 → 고신뢰, 하나만 → 저신뢰
