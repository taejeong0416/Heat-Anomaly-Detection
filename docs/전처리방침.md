# 전처리 방침

> Phase 1 정밀 전처리의 의사결정 근거를 정리한 문서.
> EDA 결과와 열 원격검침 도메인 조사를 바탕으로 작성.

---

## 요약

> **처리 순서**: 음수 → 결측 → 설비필터 → 극단값 → 후처리

1. 종별은 주택용/업무용/공공용/냉수용 전부 포함한다. 단, 냉수용은 활성시즌을 냉방시즌(6~9월)으로 반전한 뒤 같은 파이프라인을 태운다.
2. 음수값은 PNNL 파이프라인으로 처리한다. 총량음수 행 제거 → 인접양수 상쇄(±2시간 윈도우, Case 1/2/3) → 무상쇄 음수 NaN 변환.
3. 결측값은 하루 중 NaN 3개 이상이면 행 제거. 연속 NaN=1이면 선형보간, 연속 NaN≥2이면 행 제거.
4. 설비 단위로 관측 기간 1년 미만 → 결측률 28% 이상 → 처리비율 30% 이상 설비를 순서대로 제외한다.
5. 극단값은 종별 시간당 99.9%ile의 100배 이상이면 행 제거, 그 이하는 99.9%ile로 clip한다.
6. 위 규칙에 걸린 행은 학습 시에는 제거하고, 테스트 시에는 모델에 넣지 않고 해당 이상유형 라벨을 붙여서 별도 범주로 뺀다.
7. 정규화는 종별×활성시즌 Z-score로 8그룹(난방 3종별×난방시즌 6그룹 + 냉수용×냉방시즌 2그룹). 원본은 그대로 두고 파라미터만 저장.

## 1. 분석 대상 범위

### 1-1. 종별 범위

| 종별 | 포함 여부 | 근거 |
|------|:---------:|------|
| 주택용 | O | 8.4M행(49.8%), 설비 ~8,000대. 세대별 열량계. 온돌(바닥 복사난방) 패턴 |
| 업무용 | O | 6.5M행(38.3%). 건물 단위 열량계. HVAC 스케줄 기반 |
| 공공용 | O | 1.8M행(10.6%). 업무용과 유사하나 운영 주체(공공기관) 차이 |
| 냉수용 | **O** | 0.2M행(1.3%), 117개 설비. 지역냉방 냉수 계량. **시즌 정의를 반전**하여 난방 파이프라인 재사용. 냉방시즌(6~9월) 제로율 13.9%로 모델링에 문제 없음 |

> **냉수용 처리**: 난방 3종별과 동일한 파이프라인 적용. 시즌 정의만 반전(난방시즌 → 냉방시즌). 상세는 섹션 1-4 참조.

### 1-4. 냉수용 시즌 반전 처리

냉수용은 지역냉방 냉수 계량으로 난방과 계절 패턴이 정반대이지만, **분석 파이프라인 구조는 동일**하다.

| 구분 | 난방 3종별 (주택용/업무용/공공용) | 냉수용 |
|------|--------------------------------|--------|
| 활성시즌 | 난방시즌: 11~3월 | **냉방시즌: 6~9월** |
| 비활성시즌 | 비난방시즌: 4~10월 | **비냉방시즌: 10~5월** |
| 활성시즌 제로율 | 주택용 1.7%, 업무용 15.5% | **13.9%** |
| 비활성시즌 제로율 | 업무용 37.4%, 공공용 34.8% | **72.4%** |

**처리 방식**:
- 전처리(음수, 극단값, 결측 처리): 난방과 **동일한 로직** 그대로 적용
- 극단값 threshold: 냉수용 99.9%ile **별도 산출**
- 정규화: **냉수용·냉방시즌 / 냉수용·비냉방시즌** 2그룹 추가 (총 8그룹)
- Feature Engineering: 동일 Feature 사용, "난방시즌" → "냉방시즌"으로 해석 반전
- 모델링: 냉수용 별도 모델 (냉방시즌 데이터로 학습)
- 이상사용 유형: "비난방시즌 고사용" → **"비냉방시즌 고사용"** 등 해석 반전

**근거**: 냉방시즌(6~9월) 제로율 13.9%, 유효 데이터 약 7만 행, 117개 설비로 Isolation Forest/Autoencoder 학습에 충분. 주말 제로율(18~20%)도 업무용 건물 냉방 OFF로 정상 범위.

### 1-2. 시간 범위

- **전체 기간 유지**: 2021-01-01 ~ 2025-12-31 (5년)
- 난방시즌(11~3월) / 비난방시즌(4~10월) 구분하되 **둘 다 분석에 포함**
  - 근거: KDHC 열사용 비수기는 4/23~9/22이나, 업무용·공공용은 비난방기에도 급탕(온수) 수요 존재
  - 비난방시즌의 제로가 "정상"인지 "통신 장애"인지를 구분하는 것 자체가 이상 탐지의 일부

### 1-3. 설비 범위

- **NaN 28%+ 설비 제외**: 전체 설비의 약 0.5%
  - 근거: 설비별 결측률 분포 분석 결과, 95.6%가 5% 미만에 집중. 5~6% 부근에서 elbow(급감→완만 전환). 1% 단위 상세 분석 시 27~28% 구간에 16개가 있다가 28% 이후 0~1개로 급감 → **28%가 분포 단절점**.
  - 방법론: 결측률 분포 시각화 → elbow 확인 → 1% 단위 상세 분석으로 단절점(28%) 도출
  - 참고: Quesada et al. (2024) — 스마트미터 데이터셋에서 결측 비율 기반 설비 제외 방법론 적용
  - 시각화: `outputs/figures/phase02_nan_rate_analysis.png`, `phase02_nan_rate_breakpoint.png`
  - 적산열량계 법정 검정 유효기간이 **5년**이므로, 2021~2025 데이터 기간 중 교체 이벤트 포함 가능성 있음. 장기 결측은 교체 전후 공백일 수 있음
  - 시각화: `outputs/figures/phase02_nan_rate_distribution.png`

- **총 관측 기간 1년 미만 설비 제외**: 275개(전체의 2.75%)
  - 근거: 계절 패턴(난방/비난방)을 최소 1사이클 경험해야 "정상"을 정의할 수 있음
  - 참고: Quesada et al. (2024) — 총 길이 1년 이하 설비 제거
  - 세부: 2025년 신규 설치 220개 + 2021년 조기 제거 55개
  - 기간(일): min=3, median=218, max=356 / 관련 행 수: 60,695행
  - 종별: 주택용 156, 업무용 61, 공공용 58
  - 테스트 시: 해당 설비가 들어오면 `"기간부족"` 상태 반환 (이상/정상 판단 유보)

## 2. 음수값 처리 (Phase 1: 1~3단계)

### 2-1. 도메인 배경

열량계 검침 데이터에서 음수값이 발생하는 원인은 크게 4가지:

| 원인 | 메커니즘 | 데이터 특징 | 빈도 |
|------|----------|------------|------|
| **역류(Backflow)** | 고객측 압력 > 공급측 → 난방수 역방향 흐름. 야간 수요 감소, 배관 정비 중 발생 | 시간당 값이 소규모 음수 (-0.01 ~ -수 Gcal), 해당 행 총사용량은 양수일 수 있음 | 중간 |
| **검침기 교체/리셋** | 적산값이 0으로 초기화 → 이전 적산값과의 차이가 큰 음수 | **극단적 음수** 1회 발생 (-수만 ~ -수천만 Gcal), 특정 설비·특정 날짜에 집중 | 낮음 |
| **통신 오류/보정** | 원격검침 패킷 손실/타이밍 오류로 적산값 비순차 수신 | 간헐적 음수 + 이후 양수 스파이크 (보정) | 중간 |
| **온도센서 역전** | 비난방기에 공급온도 < 환수온도 → 온도차 음수 | 미세 음수 장기 지속, 총사용량도 소규모 음수 | 낮음 |

### 2-2. EDA에서 확인된 현황

- 음수값 포함 행: **6,604건** (전체 0.04%)
- 종별: 업무용 3,435 > 주택용 2,347 > 공공용 590 > 냉수용 232
- 크기: min=-73,039,997 Gcal (검침기 리셋), median=-105.5, max(가장 작은 음수)=-0.01
- **< -1 Gcal**: 5,972건 / **< -10 Gcal**: 4,830건
- 시간대 분포: 10시(645), 14시(670) 피크 → 검침 시간대와 관련 가능성
- 음수 행의 총사용량 중앙값: 약 0 (양수·음수 거의 반반)

### 2-3. 처리 방침

```
음수 전처리 — PNNL-24331 기반 3단계 파이프라인

1단계: 총량음수 행 제거 (PNNL Table 2-2)
  └── sum(24시간) < 0 → 행 제거
      근거: 하루 합계가 음수면 검침기 리셋 또는 심각한 오류.
            셀 단위 복원해도 원본 패턴 반영 불가.

2단계: 인접 양수 상쇄 음수 처리 (PNNL Section 2.4.1)
  └── 각 음수에 대해 ±2시간 윈도우 내 가장 큰 인접 양수 탐색:
      ├─ Case 1 (합 ≤ 0): 음수 + 양수 모두 NaN, 사이의 0값도 NaN
      ├─ Case 2 (합 > 0, 영향구간 ≤ 1시간): 합을 분배 보간
      └─ Case 3 (합 > 0, 영향구간 > 1시간): NaN (자동 운영 보수 정책)
      근거: 통신 오류/보정에 의한 음수는 인접 양수 스파이크와 쌍을 이룸.
            PNNL의 offset 알고리즘으로 에너지 보존하면서 음수 제거.

3단계: 무상쇄 음수 NaN 처리 (PNNL Section 2.4.2)
  └── 2단계에서 인접 양수 없는 잔여 음수 → NaN 변환
      근거: 상쇄 대상 없는 음수는 원인 불명. NaN으로 변환 후 결측 처리 로직에 위임.

테스트 시: 1단계 조건(총량음수)에 해당하면 "검침기리셋" 이상유형 라벨 부여.
```

> **이상유형 계층 분류**: `이상유형정의.md` 참조.
> **극단값이상**(극단 양수)은 섹션 3에서 별도 처리.

## 3. 극단값 처리 (Phase 4: 9단계)

> 음수 → 결측 → 설비필터 이후 마지막으로 적용. 정제된 데이터에서 threshold 산출.

### 3-1. 도메인 배경

- 주택용(세대 열량계): 일 사용량 수십 Gcal 수준이 자연적 상한. 아파트 1세대 난방 면적 20~40평 기준.
- 업무용/공공용(건물 열량계): 대형 건물은 수백 Gcal도 가능하나, **시간당** 수백 Gcal은 비현실적.
- 열량계 고장(과다계량): 센서 오류로 실제보다 높은 사용량이 기록되는 고장 유형 존재. 갑작스런 사용량 급증 패턴.

### 3-2. EDA에서 확인된 현황

**총사용량 기준** percentile:

| 종별 | 99%ile | 99.9%ile | 99.99%ile | max |
|------|--------|----------|-----------|-----|
| 주택용 | 35.9 | 63.4 | 101.9 | 28,167,600 |
| 업무용 | 19.2 | 51.2 | 98.6 | 11,319,150 |
| 공공용 | 21.0 | 81.8 | 159.1 | 60,454,910 |

- max 값이 수천만 Gcal → 검침기 오류 (정상 범위의 수십만 배)
- 99.9%ile과 99.99%ile 사이에서 급격한 점프 → 99.9%ile이 "정상 범위 상한"의 합리적 기준

### 3-3. 처리 방침

```
9단계: 종별 시간당 사용량 99.9%ile 기반 극단값 처리

  종별별 시간당 사용량 99.9%ile을 threshold로 산출
  → threshold × 100 이상인 극단값은 검침 오류로 간주, 행 제거
  → threshold 초과 ~ ×100 미만은 threshold로 clip
```

> **clip vs 제거 기준**: threshold의 100배 이상은 검침기 리셋/오작동과 동일한 성격이므로 행 제거. 그 이하는 정상 범위의 tail이므로 clip하여 보존.

**총사용량 기준**: 시간당 처리 후 재계산. 별도 총사용량 threshold 적용하지 않음 (시간당 처리로 충분).

## 4. 설비 단위 품질 필터 (Phase 3: 6~8단계)

> 음수 + 결측 처리 이후 적용.

```
6단계: 관측 기간 < 1년 설비 제거
  └── 근거: Quesada et al. (2024) — 계절 패턴 최소 1사이클 필요

7단계: 결측률 ≥ 28% 설비 제거
  └── 근거: 설비별 결측률 분포 시각화 분석
      27~28% 구간에 16개 → 28% 이후 0~1개 급감 (단절점)
      시각화: phase02_nan_rate_breakpoint.png

8단계: 처리 비율 ≥ 30% 설비 제거

처리비율 = (원본 시점 수 - 전처리 후 유효 시점 수) / 원본 시점 수

if 처리비율 >= 0.30:
    설비 제거
```

- 참고: Fowler et al. (2015) PNNL-24331 — 처리 비율 기반 설비 품질 필터 (원문 25%)
- 검증 결과: 30% 기준 제거 설비 312개(3.08%) 중 311개는 이미 6~7단계에서 제거됨. 실질 신규 제거 1개(0.01%). 30~99% 구간이 거의 비어있어 30%가 자연스러운 분기점.
- 시각화: `phase02_step8_processed_ratio.png`

## 5. 결측치(NaN) 처리

### 5-1. 도메인 배경

열 원격검침 AMI에서 NaN이 발생하는 원인:

| 원인 | 패턴 | 빈도 |
|------|------|------|
| **통신 장애** | 특정 시간대 1~수 시간 결측. 집중기(DCU) 장애 시 해당 구역 전체 | 가장 흔함 |
| **열량계 배터리 소진** | 특정 설비에서 장기간 전 시간대 결측 (NaN 24개) | 중간 |
| **열량계 고장** | 갑자기 발생, 교체 전까지 지속 | 중간 |
| **열량계 교체 작업** | 교체 당일 수 시간 결측 + 적산값 리셋 | 낮음 |
| **시스템 점검** | 정기적, 짧은 시간, 다수 설비 동시 | 낮음 |

### 5-2. EDA에서 확인된 현황

- NaN 포함 행: **209,894건** (전체 1.2%)
- **NaN 24개(전 시간대) 행: 92,240건** → 전체 NaN 행의 44%
- NaN 1~3개: 90,692건 (43%)
- NaN 4~23개: 26,962건 (13%)
- 설비 단위 NaN 비율: 96% 설비가 5% 미만. 30%+ 설비 45개.

### 5-3. 처리 방침

```
결측치 처리 (Phase 2: 4~5단계)

4단계: 일자 단위 NaN 개수 필터 (PNNL Section 2.3)
  └── 24시간 중 NaN ≥ 3개 → 행 제거
      근거: PNNL 원문 25%(=6개) 기준을 데이터 분포 분석으로 재산정.
            NaN 개수별 빈도 감소율 분석 결과:
            1→2→3에서 -99.5% → -81.2% → -76.4% 급감,
            3→4→5에서 -1.7%, +1.0% plateau.
            즉 "3개 이상부터 다른 패턴 시작" (elbow at 3).
      시각화: phase02_step4_nan_threshold.png
      결과: 0~2개 보간 가능 산발적 결측 (89,470일, 0.53%)
            3개 이상 시스템적 이상 (122,822일 제거, 0.73%)

5단계: 연속 NaN 길이 평가 (PNNL Section 3 Option 2)
  ├── 연속 NaN = 1 → 선형 보간
  │     근거: PNNL Section 3.1 — "only ≤ 1 hour gaps be filled"
  │     방법: pandas interpolate(method='linear', limit_direction='both')
  │     보간 후 음수 clip (음수 보간값은 물리적으로 불가능)
  │
  └── 연속 NaN ≥ 2 → 행 제거
        근거: PNNL Section 3 Option 2 원문 — 1시간 초과 연속 결측은
              패턴 전환 구간을 소실시킬 수 있어 보간 부적합.
```

### 5-4. 보간 후 후처리

- 보간된 값이 음수이면 0으로 clip (음수 보간값은 물리적으로 불가능)
- 보간 후 `총사용량` 재계산
- 보간 여부를 표시하는 flag 컬럼 추가 (추후 이상 탐지에서 보간 행을 별도 주의)

## 6. 제로값 처리

### 6-1. 도메인 배경

- 총사용량=0은 **반드시 이상이 아님**.
- 비난방시즌(4~10월) 업무용·공공용 건물은 난방 OFF가 정상 (급탕만 사용하거나 완전 미사용).
- 주택용은 비난방시즌에도 온수 사용 등으로 완전 제로가 드묾 (제로율 1.7%).
- 다만 **난방시즌 중 제로**는 열량계 고장(멈춤), 공실, 장기 외출 등을 의미할 수 있음.

### 6-2. 처리 방침

```
제로값은 제거하지 않는다.

이유:
- 제로 자체가 이상 탐지의 대상 ("장기 미사용 후 급증형" 유형)
- 비난방시즌 제로는 정상 → 난방시즌 flag로 구분하여 해석
- Feature Engineering에서 "연속 제로일 수"로 활용
```

## 7. 정규화

### 7-1. 방침

- **종별별로 분리하여 정규화** (종별 간 스케일 차이가 크므로)
- 방법: **Z-score** (mean/std 기반)
  - Min-Max 대비 극단값에 덜 민감하고, Isolation Forest·Autoencoder 입력으로 적합
  - 정규화 기준: 종별·활성시즌 그룹별 (비활성시즌 제로가 많아 전체 기간 통합 시 분포 왜곡)
  - 난방 3종별: 종별×난방시즌 (6그룹), 냉수용: 냉수용×냉방시즌 (2그룹) → **총 8그룹**
- 정규화 전 원본값 별도 보존
- 정규화 파라미터(mean, std) 저장 → Phase 6 결과 해석 시 역변환

### 7-2. 지사 반영

- 지사별 스케일 차이 존재 (EDA: 지사 간 평균 47배 차이)
- 그러나 지사별 정규화 시 **지사 내 소수 설비에서 분포 왜곡** 위험
- **결정**: 종별·난방시즌 그룹으로 정규화하고, 지사는 컨텍스트 Feature로 활용 (정규화 기준에는 미포함)
  - Phase 4 Context Model에서 지사 효과를 흡수

## 참고: 도메인 지식 요약

### 열량계 측정 원리
- 열량 = 유량 × (공급온도 - 환수온도)
- 초음파식(transit time difference) 또는 기계식(임펠러) 유량 측정
- 적산열량계 법정 유효기간 **5년** (본 데이터 기간 2021~2025과 일치 → 교체 이벤트 포함 가능)

### 한국 지역난방 공급 구조
```
열병합발전소 → 1차측 배관(115~120°C)
  → 건물 기계실 (판형 열교환기)
  → 2차측 배관 (세대 난방수/급탕수)
  → 각 세대 온돌(바닥 복사난방)
```

### 종별별 정상 사용 패턴
| 종별 | 피크 시간 | 주말 특성 | 비난방시즌 |
|------|-----------|-----------|-----------|
| 주택용 | 06~09시(기상), 17~22시(귀가) | 평일과 유사 (온돌 연속난방) | 소량 사용(온수) |
| 업무용 | 07~08시(예열), 09~18시(주간) | 사용량 현저 감소 | 급탕 또는 OFF |
| 공공용 | 업무용과 유사 | 감소 (공공시설 휴관) | 급탕 또는 OFF |

### 원격검침(AMI) 통신 방식
- KS-PLC(유선) + LTE(무선) 표준화 (2021년~)
- 통신 장애 원인: 무선 음역(지하 기계실), 집중기 장애, 배터리 소진, 전파 간섭
- 장애 시 적산값 비순차 수신 → 음수 또는 비정상 양수 발생

### 이상 유형의 실무적 의미
| 이상 유형 | 실무적 의미 | 조치 |
|-----------|------------|------|
| 갑작스런 급증 | 열량계 과다계량 고장, 배관 누수, 밸브 고장 | 현장 점검 |
| 야간 지속 사용 (업무용) | 밸브 미폐쇄, HVAC 스케줄 오류 | 에너지 낭비 점검 |
| 장기 제로 후 급증 | 열량계 멈춤 후 재작동, 공실 후 입주 | 고장 여부 확인 |
| 비난방시즌 고사용 | 열손실, 배관 누수, 설비 오작동 | 배관/열교환기 점검 |
| 주말 고사용 (업무용) | 무인 건물 가동, 에너지 낭비 | 운영 관리 점검 |
