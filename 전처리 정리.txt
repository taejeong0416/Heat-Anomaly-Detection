═══ Phase 1: 음수 처리 ═══

1단계: 일별 총사용량 음수 행 제거
   조건: sum(24시간) < 0
   처리: 행 제거
   이상 분류: B. 데이터 이상 - 검침기 리셋
   근거: PNNL Table 2-2

2단계: 인접 양수 상쇄 음수 처리 (PNNL Section 2.4.1)
   각 음수에 대해 ±2시간 윈도우 내 가장 큰 인접 양수 탐색:
   ├─ Case 1 (합 ≤ 0): 음수 + 양수 모두 NaN, 사이의 0값도 NaN
   ├─ Case 2 (합 > 0, 영향구간 ≤ 1시간): 합을 분배 보간
   └─ Case 3 (합 > 0, 영향구간 > 1시간): NaN (자동 운영 보수 정책)

3단계: 무상쇄 음수 NaN 처리 (PNNL Section 2.4.2)
   조건: 2단계에서 인접 양수 없는 음수
   처리: NaN

═══ Phase 2: 결측 처리 ═══

4단계: 일자 단위 의심 데이터 비율 필터 (PNNL Section 2.3)
   조건: 24시간 중 NaN ≥ 3개 
   처리: 일자 제거
   이상 분류: B. 데이터 이상 - 의심 데이터 3개 이상
   근거: PNNL Section 2.3 — 25% 원문 기준 데이터 적용 시각화 결과 기반 

5단계: 연속 NaN 길이 평가 (PNNL Section 3 Option 2)
   ├─ 연속 NaN = 1 → 선형 보간 (PNNL Section 3.1)
   └─ 연속 NaN ≥ 2 → 일자 제거
   이상 분류: B. 데이터 이상 — 연속 결측 2개 이상
   근거: PNNL Section 3 Option 2 원문 — "only ≤ 1 hour gaps be filled"

═══ Phase 3: 설비 단위 품질 필터 ═══

6단계: 관측 기간 < 1년 설비 제거
   근거: Quesada et al. (2024) Scientific Data

7단계: 결측률 ≥ 28% 설비 제거
   근거: 데이터 분포 시각화 분석

8단계: 처리 비율 ≥ 30% 설비 제거 (PNNL Section 2.3 응용)
   근거: PNNL 25% 가이드 + 본 데이터 분포 기반 재산정

9단계: 종 별 사용값 99.9%의 100배 이상이면 행 제거 + 100배 이하면 99.9%값으로 클리핑

[4단계 3개 보충]
