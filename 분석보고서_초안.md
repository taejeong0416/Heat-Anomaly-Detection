## I. 과제 개요 및 가설

### 분석 배경

한국지역난방공사는 전국 19개 지사, 약 10,000대의 열량계를 통해 주택·업무·공공·냉수용 설비의 시간별 열 사용량을 원격 검침하고 있다. 그러나 현재 이상사용 탐지는 **민원 접수 또는 정기 현장 점검에 의존**하여, 다음과 같은 행정적·사회적 문제가 발생한다.

| 문제 | 현황 | 영향 |
|------|------|------|
| **탐지 지연** | 배관 누수·밸브 고장이 민원 접수 전까지 수일~수주 방치 | 열손실 누적, 요금 분쟁 |
| **점검 비효율** | 약 10,000대 전수 점검 불가능, 점검 우선순위 부재 | 인력·비용 낭비 |
| **유형 미분류** | 이상 발견 후에도 원인(누수/밸브/센서/HVAC)을 현장 방문 전에 알 수 없음 | 대응 지연, 재방문 |
| **에너지 낭비** | 업무용 건물 야간·주말 HVAC 미해제, 기저 유량 방치 | 탄소 배출 증가 |

본 분석은 5년치(2021~2025) 열 원격검침 시간별 데이터(**16,868,714행, 10,126개 설비**)를 활용하여, 이상사용 유형을 데이터 기반으로 정의하고 자동 탐지 알고리즘을 개발함으로써, **사후 민원 대응에서 선제적 이상 탐지 체계로의 전환**을 목표로 한다.

### 핵심 가설

**주 가설**: "열 원격검침 시간별 사용 패턴에는 종별(주택/업무/공공/냉수)·시즌(난방/비난방)·요일에 따른 **정상 프로토타입**이 존재하며, 이로부터의 이탈을 정량화하면 기존 민원 기반 대비 이상사용 탐지 시점을 **수일~수주 단축**할 수 있다."

**검증 가능한 하위 가설**:

| # | 가설 | 검증 방법 | 예상 근거 |
|---|------|----------|----------|
| H1 | GMM 정상 패턴 프로토타입을 구축하면, 전체 설비-일의 **1~5%** 가 log-likelihood 하위 이상으로 식별된다 | GMM predict → LL 분포 분석 | 빌딩 에너지 이상 탐지 선행연구(Fan et al. 2018)에서 이상 비율 1~3% |
| H2 | IF+AE+MP 3종 앙상블 다수결을 적용하면, 단일 모델 대비 **Precision이 20%p 이상 향상**된다 | weak label 기반 Precision 비교 | 앙상블 다수결은 독립 분류기의 오탐을 교차 필터링(Aggarwal 2017) |
| H3 | 도메인 규칙 7유형 분류를 적용하면, 탐지된 이상의 **80% 이상**에 실무 조치 방향을 자동 매핑할 수 있다 | 유형 미분류(기타) 비율 측정 | 7유형이 도메인 조사 기반 주요 원인을 포괄 |
| H4 | 이상 탐지 자동화를 도입하면, 전수 점검 대비 우선 점검 대상을 **90% 이상 축소**할 수 있다 | 이상 설비 수 / 전체 설비 수 | 이상 비율 1~5% 가정 시 점검 대상 95~99% 축소 |

### 데이터 전처리

미국 에너지부(DOE) 산하 Pacific Northwest National Laboratory의 빌딩 에너지 데이터 품질 관리 표준(**PNNL-24331, Fowler et al. 2015**)을 국내 지역난방 데이터에 맞게 적용한 **9단계 파이프라인**:

```
[원본 16,868,714행]
  → 1단계: 총량음수 행 제거 (-1,811행)
  → 2단계: 인접양수 상쇄 (±2h 윈도우, Case1/2/3)
  → 3단계: 무상쇄 음수 → NaN (192셀)
  → 4단계: NaN≥3 일자 제거 (-122,828행)
  → 5단계: 연속NaN≥2 제거 / 연속1 보간 (-13,920행 / +75,564셀)
  → 6단계: 관측 1년 미만 설비 제거 (-279설비)
  → 7단계: 결측률 28%+ 설비 제거 (-29설비)
  → 8단계: 처리비율 30%+ 설비 제거 (-1설비)
  → 9단계: 종별 99.9%ile×100 이상 행 제거 + clip (-235행, clip 332,806셀)
[최종 16,650,282행, 9,814설비 — 원본 대비 98.7% 유지]
```

### 파생 변수(Feature) 설계

학술 문헌 기반으로 열 사용 이상을 포착하기 위한 **38개 신규 피처**를 설계하였다.

| 카테고리 | Feature | 설계 의도 | 학술 근거 |
|---------|---------|----------|----------|
| **형태** | hour_ratio_1~24 (24개) | 사용량 스케일에 무관한 시간대별 패턴 비교 | Tureczek et al. (2018) |
| **변동성** | cv, hourly_std | 하루 내 사용량 변동 크기 — 불규칙 사용 탐지 | Capozzoli et al. (2018) |

## III. 분석 방법 및 모델링

### 방법론 타당성

| 선정 기준 | 채택 방법론 | 대안 검토 및 기각 사유 |
|----------|-----------|---------------------|
| **라벨 부재** | 비지도 학습 (Unsupervised) | 지도학습: 이상 라벨 없어 적용 불가. 반지도학습: 정상 라벨도 불확실하여 부적절 |
| **다양한 이상 유형** | 3종 앙상블 (IF+AE+MP) | 단일 모델: 각 모델이 포착하는 이상 패턴이 다르므로(밀도/복원/시계열) 앙상블이 적합 |
| **해석 가능성** | 도메인 규칙 유형 분류 + SHAP | Black-box 탐지만으로는 실무 조치 불가 → 유형 분류로 "왜 이상인가" 설명 |
| **정상 패턴 정의** | GMM 클러스터링 | K-Means: 구형 클러스터만 허용, 확률적 할당 불가. DBSCAN: 고차원(24D) 비효율 |
| **스케일링** | RobustScaler (중위수/IQR) | StandardScaler: 극단값에 민감하여 일반 행이 0 근처로 압축됨 |

### 분석 프로세스 및 인과 추론 프레임워크

본 분석은 단순 통계적 이상치 검출이 아닌, **"정상 패턴으로부터의 이탈 → 이탈 유형 분류 → 도메인 원인 추론"** 3단계 인과 추론 프레임워크를 적용한다.

```
┌─────────────────────────────────────────────────────────────┐
│  1단계: 반사실적 정상 모델 (Counterfactual Normal Model)       │
│  "이 설비가 정상이었다면 어떤 패턴이었을까?"                     │
│                                                             │
│  ├─ GMM: 종별×시즌 8그룹별 정상 패턴 프로토타입 (hour_ratio)   │
│  │  → log-likelihood = "정상 패턴 적합도"                     │
│  └─ Context Z-score: 종별×지사×월×요일 그룹 중위수 대비 편차   │
│     → "같은 조건의 다른 설비 대비 얼마나 다른가"                │
└────────────────────────┬────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│  2단계: 다관점 이상 탐지 (Multi-perspective Detection)         │
│  "정상 모델에서 벗어난 정도를 독립적으로 측정"                    │
│                                                             │
│  ├─ Isolation Forest: 다차원 피처 공간에서 고립 용이성          │
│  ├─ Autoencoder: 24시간 패턴 복원 오차                        │
│  ├─ Matrix Profile: 자기 과거 시계열 대비 이질성               │
│  └─ 통계: Z-score, IQR 기반 보조 필터                         │
│                                                             │
│  앙상블: 3종 중 2개 이상 이상 판정 → 고신뢰 이상               │
└────────────────────────┬────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│  3단계: 도메인 인과 추론 (Domain Causal Reasoning)             │
│  "이상의 원인은 무엇이고, 어떤 조치가 필요한가?"                 │
│                                                             │
│  ├─ 7유형 규칙: 이상 패턴 → 도메인 원인 매핑                   │
│  │   (급증→누수/밸브, 야간→HVAC오류, 계절역행→열손실 등)       │
│  ├─ SHAP: 이상 판정에 기여한 피처/시간대 식별                   │
│  └─ 수동 검토: 유형별 대표 사례 검증                            │
└─────────────────────────────────────────────────────────────┘
```

이 프레임워크의 인과 추론 논리:
- **반사실적 비교**: "이 설비가 정상이었다면"의 기댓값(GMM 클러스터 중심, 그룹 중위수)과 실제 관측값의 차이를 이상 점수로 정량화
- **다관점 교차 검증**: 밀도 기반(IF) · 복원 기반(AE) · 시계열 기반(MP) 3가지 독립 관점에서 이상을 확인하여 단일 모델의 편향을 상쇄
- **원인 방향 추론**: 이상 패턴의 특성(야간 집중, 기저 유량, 계절 역행 등)을 도메인 지식과 매칭하여 물리적 원인(밸브 고장, 배관 누수, HVAC 오류)을 추론

### 모델링 상세

#### (1) GMM 정상 패턴 프로토타입

정상 사용 패턴을 확률적으로 정의하기 위해 **Gaussian Mixture Model**을 종별×시즌 8그룹에 적용하였다.

- **입력**: hour_ratio_1~24 (24차원) — 사용량 스케일에 무관한 시간대별 비율
- **하이퍼파라미터**: covariance_type='full', reg_covar=1e-6
- **모델 선택**: BIC(Bayesian Information Criterion) 기반 k=2~8 탐색

| 그룹 | 전체 행 | 유효 행 (제로 제외) | 최적 k | LL 평균 |
|------|--------:|------------------:|:------:|--------:|
| 주택용_난방시즌 | 3,450,708 | 3,407,634 | 8 | 87.1 |
| 주택용_비난방시즌 | 4,885,616 | 4,815,802 | 8 | 60.8 |
| 업무용_난방시즌 | 2,620,872 | 1,640,406 | 8 | 64.1 |
| 업무용_비난방시즌 | 3,729,647 | 2,383,438 | 8 | 56.5 |
| 공공용_난방시즌 | 726,377 | 513,179 | 8 | 58.2 |
| 공공용_비난방시즌 | 1,026,524 | 642,290 | 8 | 58.0 |
| 냉수용_냉방시즌 | 70,486 | 60,794 | 2 | 52.3 |
| 냉수용_비냉방시즌 | 140,052 | 38,785 | 4 | 48.7 |

- **활용**: log-likelihood → 패턴 이탈 점수 (값이 낮을수록 정상 패턴에서 먼 이상), cluster_id → 정상 프로토타입 할당
- **제로 사용일**: cluster_id=-1, gmm_log_likelihood=NaN 처리 (비난방시즌 제로는 정상)

#### (2) Context Z-score (횡단면 비교)

"같은 조건(종별·지사·월·요일)의 다른 설비들 대비 얼마나 다른가"를 측정한다.

- 그룹: **종별 × 지사 × 월 × 요일** (약 10,000+ 그룹)
- 산식: `context_zscore = (실제 총사용량 - 그룹 중위수) / 그룹 MAD`
- 중위수/MAD 사용 이유: 이상값에 robust (평균/표준편차는 이상값에 오염)
- 양수 = 과다 사용, 음수 = 과소 사용

#### (3) Isolation Forest

- 종별별 별도 학습 (종별 간 스케일·패턴 차이)
- n_estimators=200, max_samples=256
- 입력 피처: 24시간 비율 + 파생 피처 + context_zscore (**gmm_log_likelihood 제외** — 정보 누수 방지)
- contamination 그리드 탐색: 0.01 / 0.02 / 0.03 / 0.05 → weak label 기반 F1 최대화로 선정

- 이상 판정: reconstruction error 상위 N% (그리드 탐색: 1% / 2% / 3% / 5%)

#### (5) Matrix Profile (시계열 자기 참조)

기존 방법(GMM/IF/AE)이 **"다른 설비·정상 패턴 대비"** 횡단면 비교인 반면, Matrix Profile은 **"자기 자신의 과거 대비"** 종단면 비교라는 차별화된 관점을 제공한다.

- 라이브러리: stumpy (Yeh et al. 2016 알고리즘 구현)
- 설비별 일별 총사용량 시계열 구축 (설비당 최대 ~1,826일)
- Window size = 7 (주간 패턴 반복 주기)
- mp_score = nearest neighbor distance (값이 클수록 과거에 없던 이질적 구간)
- 상수 시계열(std≈0), 14일 미만 설비 → 스킵

#### (7) 9유형 전부 규칙 기반 (게이트 없음)

9개 이상사용 유형을 **전부 규칙 기반**으로 전체 데이터에 적용한다. Phase 5 `is_anomaly` 게이트를 두지 않으며, 각 규칙의 threshold(q90~q99, 3σ)가 충분히 엄격하여 별도 필터 불필요. 이상탐지는 recall 우선 원칙에 따라 설계.

**9유형 규칙**:
- 급증형: ma7_ratio + context_zscore 분위 조건
- 계절역행형: 비활성시즌 총사용량 종별 분위 초과
- 주말이상형: 업무·공공용 주말·공휴일 평일 수준 이상
- 기저유량이상형: baseload 종별·시즌 분위 초과
- 장기미사용후급증형: 설비별 직전 7일 연속 0 + 당일 급증
- 연속가동형: hourly_std 하위 + baseload 상위
- 간헐사용형: cv·hourly_std 상위

- 야간이상형: night_ratio σ 기반 (주택 +3σ, 나머지 +2σ)
- 패턴이탈형: gmm/ae/mp 점수 기반 + is_anomaly==1 잔여 catch-all

→ 9유형 전부 전체 데이터에 적용. 각 threshold(q90~q99, 3σ)가 충분히 엄격하므로 별도 게이트 불필요. 이상탐지는 recall 우선 원칙.

### 이상사용 유형 정의 (9유형, 규칙 기반)

도메인 조사(열량계 원리, 배관 물리, HVAC 운영) 기반으로 **9가지 이상사용 유형**을 정의하고, 각 유형에 물리적 원인과 실무 조치를 매핑하였다. **9유형 전부 규칙 기반으로 전체 데이터에 적용**하며, `is_anomaly==1`이지만 어떤 규칙에도 해당하지 않는 행은 패턴이탈형(catch-all)으로 분류한다.

| # | 유형 | 적용 범위 | 탐지 규칙 | 물리적 원인 | 실무 조치 |
|---|------|----------|----------|-----------|----------|
| 1 | **장기미사용후급증형** | 전체 | 설비별 직전 7일 연속 0 AND ma7_ratio 상위 20% | 열량계 재작동, 공실 후 입주 | 열량계 작동 이력 확인 |
| 2 | **계절역행형** | 전체 | 비활성시즌 총사용량 종별 분위 초과 (주택 99%, 나머지 95%) | 배관 보온재 파손, 열교환기 고장 | 배관/열교환기 점검 |
| 3 | **야간이상형** | 전체 | night_ratio ≥ 종별·월 평균 + 종별별 σ (주택 +3σ, 나머지 +2σ) | 밸브 미폐쇄, HVAC 야간 오류 | HVAC 스케줄 점검, 밸브 확인 |
| 4 | **주말이상형** | 전체 | 업무·공공용 주말·공휴일 사용량 ≥ 정상 평일 중위수 × 80% | 무인 건물 HVAC 미해제 | 운영 관리 점검, HVAC 자동화 |
| 5 | **급증형** | 전체 | ma7_ratio 상위 10% AND \|context_zscore\| 상위 10% | 배관 누수, 밸브 고장, 과다계량 | 열량계 교체 또는 배관 점검 |
| 6 | **기저유량이상형** | 전체 | baseload ≥ 종별·활성시즌 95%분위 | 배관 미세 누수 (24시간 기저 유량) | 배관 누수 점검 |
| 7 | **연속가동형** | 전체 | hourly_std ≤ 정상 5%분위 AND baseload ≥ 정상 80%분위 | 보일러 상시 가동, HVAC 24시간 | 설비 가동 스케줄 점검 |
| 8 | **간헐사용형** | 전체 | cv ≥ 정상 95%분위 AND hourly_std ≥ 정상 95%분위 | 열량계 간헐 결함, 비정형 거주 | 열량계 정밀 점검 |
| 9 | **패턴이탈형** (catch-all) | 전체 | gmm 하위 5% AND (ae 상위 5% OR mp 상위 5%) + is_anomaly==1 잔여 흡수 | 설비 오작동, 약한 비전형 신호 | 설비 종합 진단 |

> **적용 범위**: 9유형 전부 "전체" 데이터에 규칙 적용. 패턴이탈형의 catch-all(is_anomaly==1 잔여)만 모델 판정 참조.
>
> **냉수용 시즌 반전**: 냉수용은 활성시즌=냉방시즌(6~9월)으로 정의. 계절역행형은 "비냉방시즌(10~5월) 고사용", 장기미사용후급증형은 "냉방시즌 내 연속 제로 → 급증"으로 적용.

| **원인 사전 진단** | 현장 방문 후에야 원인 파악 | 7유형 자동 분류 + SHAP 기여 피처 → 점검 전 원인 가이드 제공 | **재방문율 감소, 점검 시간 단축** |
| **점검 주기** | 전 설비 동일 주기 | 이상 빈도 기반 차등 주기 (고빈도 설비 집중 관리) | **한정된 인력의 효율적 배분** |

### 리스크 관리

| 리스크 | 영향 | 대응 시나리오 |
|--------|------|-------------|
| **오탐 (False Positive)** | 불필요한 점검 출동, 현장 인력 신뢰 저하 | ① 3종 앙상블 다수결로 구조적 오탐 감소 ② 초기 6개월간 고신뢰 이상만 알림 + 수동 검토 병행 ③ 오탐 피드백 반영하여 임계치 반기 재조정 |
| **미탐 (False Negative)** | 실제 이상이 방치됨 | ① 저신뢰 이상(1/3 모델)도 모니터링 대시보드에 표시 ② 기존 민원·정기점검 체계는 병행 유지 (완전 대체 아닌 보완) |
| **민원** | "자동 시스템이 이상 판정했는데 정상이다"라는 항의 | ① 이상 판정은 "점검 권고"이지 확정 판정이 아님을 명시 ② SHAP 기반 판정 근거를 투명하게 제공 |
| **법적 제약** | 개인정보 보호, 설비 소유자 동의 | ① 분석 데이터 전체 비식별화 완료 (지사·설비 코드 익명화) ② 탐지 결과도 설비 단위 익명 코드로 제공 |
| **AI 편향성** | 특정 종별/지사에 이상이 집중 판정 | ① 종별×지사×시즌 분리 모델링으로 그룹 간 스케일 차이 제거 ② 비난방시즌 제로 사용량은 정상으로 인정 ③ 지사별 이상 비율 모니터링으로 편향 사후 감시 |
| **모델 노후화** | 신규 설비·패턴 변화로 성능 저하 | ① 정규화 파라미터 + GMM + IF/AE 모델 **연 1회 재학습** 권고 ② 월별 이상 비율 추이 모니터링 → 급변 시 조기 재학습 |
