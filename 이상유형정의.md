## 2. 데이터 이상 유형 (규칙 기반, 우선순위 계층)

우선순위가 높은 조건에 먼저 걸리면 해당 유형으로 분류하고, 이후 조건은 검사하지 않는다.

| 단계 | 유형명 | 조건 | 의미 | 학습 시 처리 |
|:----:|--------|------|------|:------------|
| 1 | **총량음수** | `sum(24h) < 0` | 검침기 리셋/심각한 오류 | 행 제거 |
| 4 | **데이터수집이상** | `nan_count >= 3` | 다수 시간대 결측, 데이터 미수신 | 행 제거 |
| 5 | **연속결측이상** | `max_consec_nan >= 2` | 연속 결측으로 패턴 복원 불가 | 행 제거 |
| 9 | **극단값이상** | `any(hour > threshold × 100)` | 물리적으로 불가능한 극단 양수 | 행 제거 |

> **threshold**: 종별별 시간당 사용량 99.9%ile (정제된 데이터에서 산출)
> 2~3단계(인접양수 상쇄, 무상쇄 NaN)는 값 보정이므로 이상유형 라벨 없이 처리

### 판정 로직 (pseudo-code)

```python
def classify_data_quality(row, thresholds):
    """PNNL 기반 데이터 이상 분류. 해당 유형을 반환."""
    hours = row[HOUR_COLS]
    
    # 1단계: 총량음수 (검침기리셋 포함)
    if hours.sum(skipna=True) < 0:
        return "총량음수"
    
    # 4단계: 데이터수집이상
    if hours.isna().sum() >= 3:
        return "데이터수집이상"
    
    # 5단계: 연속결측이상
    if max_consecutive_nan(hours) >= 2:
        return "연속결측이상"
    
    # 9단계: 극단값이상
    th = thresholds[row['종별']]
    if (hours > th * 100).any():
        return "극단값이상"
    
    return None  # 데이터 이상 아님 → 잔여 처리 후 모델 입력
```

### 학습/테스트 일관성

| 상황 | 처리 |
|------|------|
| 학습 시 | 해당 조건 행을 **제거** (모델이 깨끗한 데이터만 학습) |
| 테스트 시 | 해당 조건 행에 **이상유형 라벨 부여** (모델에 넣지 않음) |

잔여 처리(PNNL 인접양수 상쇄, 무상쇄 음수 NaN → 보간)는 학습/테스트 모두 동일하게 적용.

## 3. 이상사용 유형 (모델 기반, 9유형 — 2026-05-13 확정)

데이터 이상 필터를 통과한 데이터에 대해, 정상 패턴 모델로 탐지.
Phase 4 GMM 클러스터 검토 + 4-agent review + Phase 6b 미분류 검토 결과를 반영하여 9유형 확정.
**`is_anomaly==1` 행은 반드시 하나의 유형에 분류된다 (미분류 0%).**

### 3-0. 우선순위 (재정렬 2026-05-13)

복수 유형 동시 해당 시 아래 순서로 주 유형(`primary_type`) 결정, 나머지는 `secondary_types`.

```
장기미사용후급증 > 계절역행 > 야간이상 > 주말이상 > 급증 >
            기저유량 > 연속가동 > 간헐사용 > 패턴이탈 (catch-all)
```

> **변경 사유**: 기존(패턴이탈 1순위)에선 패턴이탈이 50~70% 흡수해 다른 유형이 가려짐. 구체적 도메인 규칙을 앞에 두고, 패턴이탈은 **catch-all**로 배치.
> 패턴이탈형은 (1) 명시 규칙(gmm 하위 5% AND (ae OR mp 상위 5%))에 부합하거나 (2) 다른 유형 어디에도 안 걸린 잔여 이상을 모두 흡수한다.

### 3-0a. 하이브리드 게이트 (2026-05-14 확정)

9유형을 **규칙 기반**과 **모델 의존**으로 구분하여 적용 범위를 달리한다.

| 범주 | 유형 | 적용 범위 | 근거 |
|------|------|----------|------|
| **규칙 기반** (7유형) | 급증, 계절역행, 주말이상, 기저유량, 장기미사용후급증, 연속가동, 간헐사용 | **전체 데이터** | 명시적 조건(분위, 시계열, σ 등)만으로 판정. Phase 5 모델 판정 불필요 |
| **모델 의존** (2유형) | 야간이상, 패턴이탈(catch-all) | **is_anomaly==1만** | 야간이상은 모델 탐지와 결합해야 오탐 억제. 패턴이탈은 모델 점수(gmm/ae/mp) 기반 + catch-all |

**설계 의도**:
- 기존: `is_anomaly==1` 게이트 → 전 유형이 Phase 5 모델에 종속 → 모델이 놓치면 규칙도 무력화 (Recall 0.640)
- 변경: 규칙 기반 유형은 모델 없이도 독립 탐지 → **Recall 개선**, Precision은 규칙 자체의 엄격성으로 유지
- 모델 의존 유형은 게이트 유지 → 야간이상/패턴이탈의 Precision 보존 (0.985 수준)

### 3-1. 9유형 정의

| # | 유형 | 판정 규칙 | 데이터·도메인 근거 |
|---|------|----------|------------------|
| 1 | **장기미사용후급증형** | `활성시즌` & **설비별 직전 7일 연속 총사용량=0** & 당일 `ma7_ratio` 상위 20% | 도메인: 열량계 멈춤 후 재작동, 공실 후 입주 |
| 2 | **계절역행형** | 비활성시즌 & 총사용량이 **종별별 임계** 이상<br>– 주택용: 99%분위<br>– 업무·공공·냉수용: 95%분위 | 도메인: 열손실/배관 누수<br>데이터: 종별 절대값 차이로 주택용 95%는 과탐 |
| 3 | **야간이상형** | `night_ratio` ≥ 종별·월 정상 평균 + **종별별 σ**<br>– 주택용: +3σ<br>– 업무·공공·냉수용: +2σ | 도메인: 밸브 미폐쇄, HVAC 스케줄 오류<br>데이터: Phase 4 주택용 C0(34.9%) 새벽 가동 정상 클러스터 회피 |
| 4 | **주말이상형** | 업무용/공공용 & 주말·공휴일 & 총사용량 ≥ 정상 평일 중위수 × 0.8 | 도메인: 무인 건물 HVAC 미해제 |
| 5 | **급증형** | `ma7_ratio` 상위 10% AND \|`context_zscore`\| 상위 10% | 도메인: 배관 누수, 밸브 고장, 과다계량 |
| 6 | **기저유량이상형** | `baseload` ≥ 종별·활성시즌 95%분위 | 도메인: 배관 누수 |
| 7 | **연속가동형** | `hourly_std` ≤ 종별·활성시즌 정상 5%분위 AND `baseload` ≥ 정상 80%분위 | 데이터: Phase 4 주택 C4(41.6%) 평탄 클러스터의 더 극단판<br>도메인: 보일러 상시 가동, 누수 지속 유량 |
| 8 | **간헐사용형** (신규) | `cv` ≥ 종별·활성시즌 정상 95%분위 AND `hourly_std` ≥ 정상 95%분위 | 데이터: Phase 6b 미분류 검토 — cv z=+2.01, ae z=+3.11 (들쭉날쭉 패턴)<br>도메인: 열량계 간헐 결함, 비정형 거주 패턴, 급수 결함 |
| 9 | **패턴이탈형** (catch-all) | 명시 규칙: `gmm_log_likelihood` 하위 5% AND (`ae_score` 상위 5% OR `mp_score` 상위 5%)<br>**+ is_anomaly==1 잔여(모든 구체 유형 미부합)** | 도메인: 설비 오작동, 약한 비전형 신호<br>역할: catch-all로 **미분류 0% 보장** |

### 3-2. 핵심 의사결정 (2026-05-13 재검토 + 2차 보강)

| 변경 항목 | 이전 | 변경 후 | 사유 |
|-----------|------|---------|------|
| **장기미사용 정의** | `zero_count ≥ 7` (24시간 중 0인 시간 수) | **설비별 직전 7일 연속 총사용량=0** | 원 정의("연속 N일 미사용")와 구현 불일치 — 시계열 롤링으로 재정의 |
| **계절역행 임계** | 전 종별 95% | 주택 99% / 나머지 95% | 주택용 절대값이 작아 과탐 |
| **야간이상 적용 종별·σ** | 전 종별 +2σ | 주택 +3σ, 나머지 +2σ | Phase 4 주택용 C0(34.9%) 새벽 가동 정상 클러스터 회피하되, 주택용도 적용 복원 |
| **패턴이탈 임계** | gmm 하위 10% OR ae 상위 10% OR mp 상위 10% | gmm 하위 5% AND (ae 상위 5% OR mp 상위 5%) | OR 3중 결합이 50~70% 흡수 → AND 결합 + fallback 적정 흡수를 위해 5% |
| **우선순위** | 패턴이탈 1순위 | 패턴이탈 마지막(fallback) | 광범위 패턴이탈이 구체 유형 가림 |
| **신규 유형 (F)** | — | **연속가동형 추가** | 데이터: Phase 4 주택 C4(41.6%) 평탄 클러스터의 더 극단판이 미분류로 누락. 도메인: 보일러 상시 가동 / 누수 |
| **신규 유형 (I)** | — | **간헐사용형 추가** | 데이터: Phase 6b 미분류 50건 검토 결과, 비활성시즌에 들쭉날쭉 패턴(cv z=+2, ae z=+3) 다수. 도메인: 열량계 간헐 결함 / 비정형 거주 / 급수 결함 |
| **catch-all 정책** | 미분류 35% 발생 | **패턴이탈형이 잔여 흡수 → 미분류 0%** | Phase 6b 검토 후 결정: 잔여 미분류는 약한 비전형 신호이므로 패턴이탈로 통합 |

### 3-3. 냉수용 시즌 반전 적용

냉수용은 난방과 계절 패턴이 정반대이므로, **이상사용 유형의 시즌 기준을 반전**하여 적용한다.

| 이상사용 유형 | 난방 3종별 (주택용/업무용/공공용) | 냉수용 |
|-------------|--------------------------------|--------|
| 계절역행형 | 비난방시즌(4~10월) 고사용 | **비냉방시즌(10~5월) 고사용** |
| 장기미사용후급증 | 난방시즌(11~3월) 내 7일 연속 0 → 급증 | **냉방시즌(6~9월) 내 7일 연속 0 → 급증** |
| 기타 유형 | 동일 로직 (급증, 야간, 패턴 이탈 등) | **동일 로직** |

> **파이프라인 구현**: 종별에 따라 `활성시즌` 플래그를 참조. 난방 3종별은 `난방시즌`, 냉수용은 `냉방시즌`을 사용.

## 4. 출력 스키마

최종 이상 탐지 결과는 설비-일 단위로 아래 형태를 목표로 한다.

```
설치     | 날짜       | 이상유형       | 탐지근거          | 심각도
---------|-----------|---------------|------------------|-------
A001     | 2024-01-15 | 검침기리셋     | hour_3=-73M      | -
A001     | 2024-02-10 | 데이터수집이상  | nan_count=24     | -
B042     | 2024-03-05 | (모델 유형)    | anomaly_score=3.2 | 0.85
```

- 데이터 이상: 심각도 불필요 (규칙에 걸리면 확정)
- 이상사용: 심각도 점수 (0~1, 모델 이상 점수 정규화)
