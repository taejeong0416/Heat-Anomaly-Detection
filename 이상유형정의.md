## 2. 데이터 이상 유형 (규칙 기반, 우선순위 계층)

우선순위가 높은 조건에 먼저 걸리면 해당 유형으로 분류하고, 이후 조건은 검사하지 않는다.

| 우선순위 | 유형명 | 조건 | 의미 | 학습 시 처리 |
|:--------:|--------|------|------|:------------|
| 1 | **검침기리셋** | `any(hour < -10,000)` | 적산값 초기화로 추정되는 극단 음수 | 행 제거 |
| 2 | **총량음수** | `total < 0` | 하루 전체 사용량이 음수, 전체 오염 | 행 제거 |
| 3 | **극단값이상** | `any(hour > threshold × 100)` | 물리적으로 불가능한 극단 양수 | 행 제거 |
| 4 | **다중음수이상** | `neg_count >= 3` | 3개+ 시간대에서 음수, 광범위 오염 | 행 제거 |
| 5 | **데이터수집이상** | `nan_count >= 3` | 다수 시간대 결측, 데이터 미수신 | 행 제거 |

> **threshold**: 종별별 시간당 사용량 99.9%ile (학습 데이터에서 산출, `norm_params.json` 또는 별도 저장)

### 판정 로직 (pseudo-code)

```python
def classify_data_quality(row, thresholds):
    """우선순위 계층 필터. 먼저 걸린 유형을 반환."""
    hours = row[HOUR_COLS]
    
    # 1순위: 검침기리셋
    if (hours < -10000).any():
        return "검침기리셋"
    
    # 2순위: 총량음수
    if row['총사용량'] < 0:
        return "총량음수"
    
    # 3순위: 극단값이상
    th = thresholds[row['종별']]
    if (hours > th * 100).any():
        return "극단값이상"
    
    # 4순위: 다중음수이상
    if (hours < 0).sum() >= 3:
        return "다중음수이상"
    
    # 5순위: 데이터수집이상
    if hours.isna().sum() >= 3:
        return "데이터수집이상"
    
    return None  # 데이터 이상 아님 → 잔여 처리 후 모델 입력
```

### 학습/테스트 일관성

| 상황 | 처리 |
|------|------|
| 학습 시 | 해당 조건 행을 **제거** (모델이 깨끗한 데이터만 학습) |
| 테스트 시 | 해당 조건 행에 **이상유형 라벨 부여** (모델에 넣지 않음) |

잔여 처리(경미한 음수 클리핑, 큰 음수 결측→보간)는 학습/테스트 모두 동일하게 적용.

## 3. 이상사용 유형 (모델 기반)

데이터 이상 필터를 통과한 데이터에 대해, 정상 패턴 모델로 탐지.

> Phase 5~6에서 모델 결과 기반으로 세부 유형 정의 예정.
> 예상되는 유형 후보 (도메인 기반):

| 유형 후보 | 패턴 | 실무적 의미 |
|----------|------|------------|
| 갑작스런 급증 | 평소 대비 사용량 급등 | 배관 누수, 밸브 고장, 열량계 과다계량 |
| 야간 지속 사용 | 업무용 야간/주말 비정상 사용 | 밸브 미폐쇄, HVAC 스케줄 오류 |
| 장기 미사용 후 급증 | 연속 제로 → 갑자기 고사용 | 열량계 고장 후 재작동, 공실 후 입주 |
| 비난방시즌 고사용 | 4~10월 이상 사용 | 열손실, 배관 누수, 설비 오작동 |

> 최종 유형은 모델 출력(이상 점수, 클러스터링 등)에 기반하여 확정.

## 4. 출력 스키마

최종 이상 탐지 결과는 설비-일 단위로 아래 형태를 목표로 한다.

```
설치     | 날짜       | 이상유형       | 탐지근거          | 심각도
---------|-----------|---------------|------------------|-------
A001     | 2024-01-15 | 검침기리셋     | hour_3=-73M      | -
A001     | 2024-02-10 | 데이터수집이상  | nan_count=24     | -
B042     | 2024-03-05 | (모델 유형)    | anomaly_score=3.2 | 0.85
```

- 데이터 이상: 심각도 불필요 (규칙에 걸리면 확정)
- 이상사용: 심각도 점수 (0~1, 모델 이상 점수 정규화)
