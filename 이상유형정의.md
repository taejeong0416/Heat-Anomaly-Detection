## 2. 데이터 이상 유형 (규칙 기반, 우선순위 계층)

우선순위가 높은 조건에 먼저 걸리면 해당 유형으로 분류하고, 이후 조건은 검사하지 않는다.

| 단계 | 유형명 | 조건 | 의미 | 학습 시 처리 |
|:----:|--------|------|------|:------------|
| 1 | **총량음수** | `sum(24h) < 0` | 검침기 리셋/심각한 오류 | 행 제거 |
| 4 | **데이터수집이상** | `nan_count >= 3` | 다수 시간대 결측, 데이터 미수신 | 행 제거 |
| 5 | **연속결측이상** | `max_consec_nan >= 2` | 연속 결측으로 패턴 복원 불가 | 행 제거 |
| 9 | **극단값이상** | `any(hour > threshold × 100)` | 물리적으로 불가능한 극단 양수 | 행 제거 |

> **threshold**: 종별별 시간당 사용량 99.9%ile (정제된 데이터에서 산출)
> 2~3단계(인접양수 상쇄, 무상쇄 NaN)는 값 보정이므로 이상유형 라벨 없이 처리

### 판정 로직 (pseudo-code)

```python
def classify_data_quality(row, thresholds):
    """PNNL 기반 데이터 이상 분류. 해당 유형을 반환."""
    hours = row[HOUR_COLS]
    
    # 1단계: 총량음수 (검침기리셋 포함)
    if hours.sum(skipna=True) < 0:
        return "총량음수"
    
    # 4단계: 데이터수집이상
    if hours.isna().sum() >= 3:
        return "데이터수집이상"
    
    # 5단계: 연속결측이상
    if max_consecutive_nan(hours) >= 2:
        return "연속결측이상"
    
    # 9단계: 극단값이상
    th = thresholds[row['종별']]
    if (hours > th * 100).any():
        return "극단값이상"
    
    return None  # 데이터 이상 아님 → 잔여 처리 후 모델 입력
```

### 학습/테스트 일관성

| 상황 | 처리 |
|------|------|
| 학습 시 | 해당 조건 행을 **제거** (모델이 깨끗한 데이터만 학습) |
| 테스트 시 | 해당 조건 행에 **이상유형 라벨 부여** (모델에 넣지 않음) |

잔여 처리(PNNL 인접양수 상쇄, 무상쇄 음수 NaN → 보간)는 학습/테스트 모두 동일하게 적용.

## 3. 이상사용 유형 (모델 기반)

데이터 이상 필터를 통과한 데이터에 대해, 정상 패턴 모델로 탐지.

> Phase 5~6에서 모델 결과 기반으로 세부 유형 정의 예정.
> 예상되는 유형 후보 (도메인 기반):

| 유형 후보 | 패턴 | 실무적 의미 |
|----------|------|------------|
| 갑작스런 급증 | 평소 대비 사용량 급등 | 배관 누수, 밸브 고장, 열량계 과다계량 |
| 야간 지속 사용 | 업무용 야간/주말 비정상 사용 | 밸브 미폐쇄, HVAC 스케줄 오류 |
| 장기 미사용 후 급증 | 연속 제로 → 갑자기 고사용 | 열량계 고장 후 재작동, 공실 후 입주 |
| 비활성시즌 고사용 | 난방: 4~10월, 냉수: 10~5월 이상 사용 | 열손실, 배관 누수, 설비 오작동 |

> 최종 유형은 모델 출력(이상 점수, 클러스터링 등)에 기반하여 확정.

### 3-1. 냉수용 시즌 반전 적용

냉수용은 난방과 계절 패턴이 정반대이므로, **이상사용 유형의 시즌 기준을 반전**하여 적용한다.

| 이상사용 유형 | 난방 3종별 (주택용/업무용/공공용) | 냉수용 |
|-------------|--------------------------------|--------|
| 계절 역행형 | 비난방시즌(4~10월) 고사용 | **비냉방시즌(10~5월) 고사용** |
| 장기 미사용 후 급증 | 난방시즌(11~3월) 내 연속 제로 → 급증 | **냉방시즌(6~9월) 내 연속 제로 → 급증** |
| 기타 유형 | 동일 로직 (급증, 야간, 패턴 이탈 등) | **동일 로직** |

> **파이프라인 구현**: 종별에 따라 `활성시즌` 플래그를 참조. 난방 3종별은 `난방시즌`, 냉수용은 `냉방시즌`을 사용.

## 4. 출력 스키마

최종 이상 탐지 결과는 설비-일 단위로 아래 형태를 목표로 한다.

```
설치     | 날짜       | 이상유형       | 탐지근거          | 심각도
---------|-----------|---------------|------------------|-------
A001     | 2024-01-15 | 검침기리셋     | hour_3=-73M      | -
A001     | 2024-02-10 | 데이터수집이상  | nan_count=24     | -
B042     | 2024-03-05 | (모델 유형)    | anomaly_score=3.2 | 0.85
```

- 데이터 이상: 심각도 불필요 (규칙에 걸리면 확정)
- 이상사용: 심각도 점수 (0~1, 모델 이상 점수 정규화)
