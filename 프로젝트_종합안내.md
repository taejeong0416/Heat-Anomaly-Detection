# 열 원격검침 이상사용 탐지 프로젝트 종합 안내

---

## 1. 프로젝트 개요

### 배경 및 목표

국내 지역난방 공급 설비의 **열량계 원격검침 시간별 사용량** 데이터를 기반으로, **이상사용 유형을 직접 정의**하고 이를 자동 탐지하는 알고리즘을 개발한다.

### 기존 방식의 한계

| 문제점 | 설명 |
|--------|------|
| 단순 임계치 기반 | 전년·전전년 대비 ±30% 기준으로만 판단 |
| 패턴 이상 탐지 불가 | 시간대별 사용 패턴의 이상을 감지하지 못함 |
| 수작업 의존 | 시간대별 이상은 담당자가 직접 확인해야 함 |

### 개선 방향

- 다양한 이상 패턴을 **자동 탐지**
- 이상 유형별 분류 + 알고리즘 설계
- 단순 값이 아닌 **패턴 기반** 접근

### 2.3 데이터 구조 — 계층 관계

```
고객번호 (건물 단위)
  └─ 기계실 (건물 내 기계실)
       └─ 설치 (개별 열량계) ← 가장 세분화된 분석 단위
```

- 하나의 고객(건물)에 여러 기계실이 존재 가능
- 하나의 기계실에 여러 열량계(설치)가 존재 가능

### 2.4 데이터 규모 (참고)

| 기준 | 2021.01 | 2025.01 |
|------|---------|---------|
| 고객번호 | 약 5,300개 | 약 6,100개 |
| 기계실 | 약 7,000개 | - |
| 설치(열량계) | 약 8,500대 | 약 9,700대 |
| 월별 행 수 | 약 262,000행 | - |

※ 기간 중 신규 설치 및 폐지된 열량계 존재

### 2.5 데이터 특성 및 유의사항

- **라벨 없음** → 비지도 학습 필요
- 각 행 = 하나의 건물 + 특정 날짜의 **24시간 사용 패턴** (24차원 시계열 벡터)
- 사용량 0.0 다수 존재 (비난방 시즌 또는 미사용 설비)
- `NaN` = 해당 시간 검침 데이터 없음 (통신 두절 등)
- **비식별화 처리**: 종별 단위로 랜덤 가중치 적용 → 절대값은 원본과 다르나 **상대적 패턴(비율)은 보존**
- 동일 종별 열량계 간 사용량 비교는 유효
- 식별정보 코드는 전체 기간 동일 매핑 → **시계열 추적 및 그룹핑 가능**

## 3. 이상사용 유형 정의 (최소 3개 이상 직접 정의)

### (1) 급증형

- 전일 대비 또는 동일 시간대 대비 사용량 급격히 증가
- 배관 누수, 밸브 오작동 등 설비 이상 가능성

### (2) 야간 이상형

- 22시~06시 사이 비정상적 사용 발생
- 정상적으로 사용이 없어야 할 시간대에 지속적 소비

### (3) 패턴 이탈형

- 동일 종별/유형 건물의 전형적 사용 곡선에서 벗어난 형태
- 예: 업무용 건물이 주말에 평일 수준 사용

### (4) 장기 미사용 후 급증형

- 일정 기간 사용량 0 또는 극소 → 갑작스런 대량 사용
- 장비 오작동, 검침 오류 가능성

### (5) 계절 역행형

- 계절/기온 조건과 맞지 않는 사용 패턴
- 예: 한여름 난방 사용, 한겨울 냉방 사용

## 4. 분석 방법 (Pipeline)

### Step 1. 데이터 전처리

- 결측치 처리 (보간 또는 제거)
- 명백한 오류값 제거 (음수, 비현실적 값)
  - **주의**: 이상사용 패턴과 장비 오류를 구분 → 오류만 제거, 이상 패턴은 보존
- Min-Max 정규화 (종별/유형별 그룹 내)

### Step 2. Feature Engineering

| 구분 | Feature |
|------|---------|
| 기본 | 시간별 사용량 (1~24시) |
| 파생 | 총 사용량, peak 시간대, 야간 비율(22~06시/총량), 시간별 변동성(표준편차), 전일 대비 변화율 |
| 외부 변수 | 기온(기상청 종관기상관측), 요일/공휴일 여부, 종별·유형(범주형) |

### Step 3. Clustering (정상 패턴 분류)

- K-Means로 사용 패턴 군집화
- DTW(Dynamic Time Warping) 거리 기반 클러스터링 보조 활용
- 종별/유형별 정상 패턴 프로토타입 정의

### Step 4. Context Modeling (기대 사용량 예측)

- 기온, 요일, 종별 등을 입력으로 기대 사용량 예측
  - 방법: LightGBM 회귀 또는 동일 조건(종별+유형+요일+기온구간) 그룹 평균
- Residual 계산: `실제 사용량 - 기대 사용량`
- Residual이 큰 샘플 = 맥락 대비 이상

### Step 5. Anomaly Detection

**정석 기법:**

- **Isolation Forest**: 전체 feature 공간에서 global 이상치 탐지
- **통계 기반**: 종별/유형별 z-score, IQR 기반 필터링

**차별점 기법:**

| 기법 | 설명 |
|------|------|
| Autoencoder 복원 오차 | 24차원 → 저차원(8~12) 압축 → 24차원 복원. 정상: 오차 낮음 / 이상: 오차 높음. Isolation Forest와 앙상블 |
| Matrix Profile (Discord 탐지) | 건물별 시계열에서 가장 유사한 패턴이 없는 구간(discord) 자동 탐지. 라이브러리: `stumpy` |
| SHAP 기반 이상 원인 해석 | Isolation Forest 결과에 SHAP value 적용 → "몇 시 사용량이 이상 판정에 가장 기여했는지" 시각화 |

### Step 6. 이상 유형 분류 (탐지 → 유형 매핑)

| 유형 | 분류 규칙 |
|------|-----------|
| 급증형 | 전일 대비 변화율 > threshold + Residual 상위 |
| 야간 이상형 | 야간 비율이 동일 종별 평균 대비 유의하게 높음 |
| 패턴 이탈형 | 클러스터 소속 확률 낮음 또는 AE 복원 오차 높음 |
| 장기 미사용 후 급증형 | 연속 N일 미사용 → 급증 패턴 |
| 계절 역행형 | 기온과 사용 유형 간 불일치 |

### Step 7. 결과 검증

- 탐지 결과 시각화 (시간별 사용량 그래프 + 이상 구간 표시)
- 유형별 대표 사례 추출 및 해석
- SHAP 기여도 차트로 "왜 이상인지" 근거 제시
- 종별/지사별 이상 빈도 통계

## 7. 핵심 전략 요약

1. **패턴 기반 분석** — 단순 값이 아니라 시간 흐름(24차원 벡터) 분석
2. **비지도 학습 활용** — 라벨 없이 정상 패턴 학습 후 이탈 탐지
3. **외부 변수 결합** — 기온·요일 등으로 맥락 반영, 정확도 향상
4. **정석 + 차별점 조합** — Isolation Forest(정석) + Autoencoder·Matrix Profile·SHAP(차별점)으로 탐지 신뢰도와 해석력 확보
5. **모델보다 Feature와 유형 정의가 핵심** — 좋은 유형 정의와 Feature가 모델 성능을 좌우
