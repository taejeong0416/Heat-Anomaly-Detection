## 데이터 기본 정보

- 60개 월별 CSV (gzip), `data/raw/` 하위 연도별 폴더 (`data/raw/2021/`~`data/raw/2025/`)
- 인코딩: `utf-8-sig`
- 컬럼 34개: 날짜, 지사~설치(익명화), 종별, 유형, 1시~24시, 총
- 종별: 주택용 / 업무용 / 공공용 / 냉수용
- 유형: 전 기간 `-`만 존재 (사실상 사용 불가)
- 총사용량 컬럼명: `총`
- 월별 약 26만 행, 설비(설치) 약 8,500~9,700대

## 규칙

- 분석 진행 시 `분석플랜.md`의 체크리스트를 업데이트할 것
- 단계 완료 시 이 파일의 진행 상황도 함께 업데이트
- 노트북은 `notebooks/` 폴더에 번호 순서로 저장
- **각 Phase에서 중요한 의사결정이 있을 때마다 근거 문서를 남길 것** (예: `전처리방침.md`, `도메인조사.md`)
- 도메인 지식이 필요한 의사결정은 반드시 `도메인조사.md`를 참조하고, 새로운 도메인 지식이 발견되면 해당 문서에 추가

## 산출물 규칙

각 Phase 완료 후 반드시 아래 산출물을 남긴다.

### 1. 노트북 (`notebooks/`)
- 파일명: `XX_단계명.ipynb` (예: `01_preprocessing.ipynb`)
- 노트북 상단에 **목적, 입력 데이터, 출력 데이터** 명시
- 셀 단위로 마크다운 설명 포함 (코드만 나열하지 않을 것)
- 최종 실행 결과가 저장된 상태로 커밋

### 2. 데이터 (`data/processed/`)
- `data/processed/` 에는 **현재 사용 중인 최신 데이터만** 둔다
- 과거 버전, 중간 산출물, 로그 등 히스토리 보관용 파일은 `data/processed/archive/`에 저장
- 데이터는 **parquet** 형식으로 저장

### 3. 시각화 (`outputs/figures/`)
- 주요 차트는 PNG로 저장 (dpi=150 이상)
- 파일명: `phaseXX_설명.png` (예: `phase01_hourly_pattern_by_type.png`)

### 4. 단계 요약
- 각 Phase 완료 시 노트북 마지막 셀에 **핵심 발견사항 3~5줄** 요약
- 다음 Phase에 영향을 주는 의사결정 사항을 명확히 기록

### 5. Phase 완료 보고서 (프로젝트 루트)
- 각 Phase 완료 시 `PhaseN_완료보고.md` 작성 (협업프로세스 ④ 팀장 취합 보고 형식)
- **`최종보고서.md`를 매 Phase 완료 시 누적 업데이트** (진행 현황, 데이터 현황, 핵심 결과, 의사결정 이력, 리스크, 산출물 등)
- 완료 보고서에는 반드시 포함: 핵심 결과, 처리 상세, 토론 쟁점/결론, 다음 Phase 영향, 리스크, 산출물 목록

### 6. 의사결정 문서 (프로젝트 루트)
- 각 Phase에서 중요한 분석 방향/방법론/threshold 결정 시 **근거 문서를 별도 md로 작성**
- 파일명 예시: `전처리방침.md`, `도메인조사.md`, `이상유형정의.md`
- 도메인 지식이 필요한 의사결정은 `도메인조사.md`를 참조·업데이트
- 문서 작성 후 이 파일의 핵심 파일 표에 추가
